Image pixels and channels

Detailed explanation

A digital image is a rectangular grid of pixels. Each pixel stores numerical values representing color intensity. In a grayscale image, each pixel is a single number from 0 (black) to 255 (white). In a color image using the RGB model, each pixel is a triplet of values: Red intensity, Green intensity, Blue intensity — each from 0 to 255.

In PyTorch and most deep learning frameworks, an image tensor has shape (C, H, W): C channels, H height in pixels, W width in pixels. An RGB image of 224×224 pixels has shape (3, 224, 224). When batched, it becomes (N, C, H, W) — N images stacked along the first dimension. This convention is different from NumPy/PIL which use (H, W, C) — converting between them requires .permute() or .transpose().

Pixel values are usually normalized before being fed to a neural network. Raw values are integers 0–255. Dividing by 255 converts them to floats in [0, 1]. Subtracting the channel-wise mean and dividing by channel-wise standard deviation (computed on the training set) standardizes to zero mean and unit variance. This helps gradient flow and ensures all features are on comparable scales.

The three RGB channels are not the only option. Medical imaging uses more channels: CT scans can have many slices, satellite imagery can have infrared and multispectral channels, depth cameras produce a 4th depth channel (RGBD). The network architecture simply changes the first convolutional layer's input channels to match.

Understanding pixel representation prevents preprocessing bugs. The most common one: loading an image with PIL or OpenCV in (H, W, C) format and feeding it directly to a model expecting (C, H, W) — the model receives spatial information in the wrong dimension and produces nonsense without any error message.

Why this matters in AI
Correct image preprocessing is the first step for any vision model. Wrong channel ordering or wrong normalization silently produces bad predictions — the model sees senseless input but still outputs a confident (wrong) answer.
Common mistakes
  • Using (H, W, C) format instead of (C, H, W) when passing to PyTorch conv layers — the model treats spatial dimensions as channels, producing garbage activations.
  • Not normalizing: feeding raw 0–255 integers to a model pretrained on 0–1 normalized images — activation magnitudes are 255× larger than expected, breaking all learned features.
  • Assuming all images have 3 channels — grayscale images have 1 channel; PNG images can have 4 (RGBA). Always check img.mode and handle accordingly.

Image preprocessing

Detailed explanation

Image preprocessing transforms raw images into tensors suitable for neural network input. The standard pipeline for a pre-trained ImageNet model is: Resize → CenterCrop (or RandomCrop for training) → ToTensor → Normalize. Each step has a purpose: resize makes images a manageable size, crop ensures consistent spatial dimensions, ToTensor converts to float tensors in [0,1], normalize standardizes to the pre-trained model's expected distribution.

For training, preprocessing includes random augmentations that artificially increase dataset diversity: random horizontal flip, random crop from a slightly larger resize, color jitter (brightness, contrast, saturation, hue), and rotation. These are applied randomly to each image every epoch, so the model sees different versions of the same training image and learns to generalize. For validation and test, only deterministic transforms are used (resize, center crop, normalize) — no randomness.

Image size significantly impacts model speed and accuracy. Smaller images (112×112) train 4× faster than larger ones (224×224) with the same batch size. Pre-trained ImageNet models expect 224×224 for most architectures. EfficientNet variants use different sizes (240, 260, 300+). Always check what input size a pre-trained model expects before preprocessing.

Beyond resizing and normalization, some tasks require task-specific preprocessing. Object detection needs bounding box coordinates transformed alongside the image (same flip and crop applied to boxes). Instance segmentation needs pixel-level masks transformed consistently with the image. Medical imaging often requires DICOM-specific preprocessing like windowing (mapping HU units to a visible range) before standard normalization.

torchvision.transforms.Compose chains multiple transforms sequentially. transforms.v2 (the newer API) is recommended for tasks involving bounding boxes and masks because it applies transforms consistently across image and annotation. Always check whether your transform applies correctly to your data type.

Why this matters in AI
Preprocessing determines what information reaches the model. Inconsistent preprocessing between training and inference is one of the most common sources of production model degradation.
Common mistakes
  • Applying random augmentations at inference time — predictions become stochastic and unreliable (unless you are doing test-time augmentation intentionally).
  • Using different normalization statistics for fine-tuning vs pre-training — if the pre-trained model expects ImageNet normalization, use those exact statistics.
  • Resizing with the wrong interpolation — INTER_NEAREST for classification labels/masks, INTER_LINEAR or INTER_CUBIC for RGB images.

CNN

Detailed explanation

A Convolutional Neural Network (CNN) is a deep learning architecture designed for processing data with spatial structure — primarily images, but also audio and video. CNNs replace the fully connected layers of an MLP with convolutional layers that exploit locality (nearby pixels are more related than distant ones) and translational equivariance (a feature detector that finds a dog's ear should work regardless of where in the image the ear appears).

The core building block is the convolutional layer. It slides a small learnable filter (kernel) across the image, computing a dot product at each position. This produces a feature map: a 2D activation showing where in the image the filter detected its pattern. Multiple filters produce multiple feature maps, each detecting a different pattern. Early CNN layers learn to detect edges and colors; deeper layers learn textures, parts, and eventually full objects.

A typical CNN architecture alternates convolutional layers with pooling layers, with activation functions after each conv layer. The final layers transition to fully connected (dense) layers for classification. The famous LeNet (1998) pioneered this pattern. AlexNet (2012) won the ImageNet competition and sparked the deep learning revolution by applying this at scale on GPUs. ResNet (2015) introduced skip connections that enabled training networks 100+ layers deep.

The key advantage over MLP for images: a 224×224 RGB image has 224×224×3 ≈ 150,000 input pixels. A fully connected layer with 512 neurons would need 150,000×512 ≈ 77 million parameters just in the first layer. A convolutional layer with 32 filters of size 3×3 needs only 32×3×3×3 = 864 parameters regardless of image size, because the same filter is shared across all spatial positions.

Standard CNN building blocks in PyTorch: nn.Conv2d(in_channels, out_channels, kernel_size, stride, padding) for the convolution, nn.BatchNorm2d(out_channels) after conv, nn.ReLU() for activation, nn.MaxPool2d(kernel_size, stride) for downsampling, and nn.AdaptiveAvgPool2d((1,1)) before the classifier head to handle variable input sizes.

Why this matters in AI
CNNs are the foundation of computer vision. Every vision model — from image classifiers to object detectors to generative models — builds on convolutional blocks. Understanding how convolutions work explains why vision models are so parameter-efficient compared to MLPs.
Common mistakes
  • Not using padding in conv layers — without padding='same' or padding=1 for 3×3 kernels, feature maps shrink with every layer and deep networks collapse to tiny spatial dimensions.
  • Using global average pooling incorrectly — if your final feature map is not spatially meaningful, GAP can destroy useful spatial information.
  • Forgetting that Conv2d weight shape is (out_channels, in_channels, kH, kW) — confusing in/out channels when stacking layers.

Convolution filters

Detailed explanation

A convolution filter (also called kernel) is a small matrix of learnable weights, typically 3×3, 5×5, or 7×7. The filter slides across the input feature map, and at each position, computes the dot product between the filter weights and the local patch of the input it overlaps. This produces a single value at each position — all values together form the output feature map.

Before learning, filters can be understood intuitively with fixed hand-crafted examples. A horizontal edge detector kernel [[1,1,1],[0,0,0],[-1,-1,-1]] produces large positive values where bright pixels are above dark pixels and large negative values at the reverse edge. A blur kernel (all values = 1/9) averages each pixel with its neighbors, smoothing the image. CNNs learn analogues of these detectors automatically from data.

The filter's receptive field is the region of the input it 'sees'. A 3×3 filter in the first layer sees a 3×3 patch of pixels. After stacking two conv layers with 3×3 filters, the second layer effectively sees a 5×5 patch of the original input (because it sees 3×3 of the first layer's output, each of which saw 3×3 of the input). Deeper layers have larger effective receptive fields without increasing the kernel size.

Stride and padding control output spatial size. Stride=1 slides the filter one pixel at a time, producing output same size as input (with padding) or slightly smaller (without padding). Stride=2 slides two pixels at a time, halving the spatial dimension — a computationally efficient alternative to pooling for downsampling. Padding='same' adds zeros around the border so output matches input size for stride=1.

Depthwise separable convolutions (used in MobileNet) split a standard convolution into two steps: a depthwise convolution (one filter per input channel, no mixing across channels) followed by a pointwise 1×1 convolution (mixes channels). This reduces parameters by 8–9× for a 3×3 kernel while maintaining similar accuracy — making models much more efficient for mobile and edge deployment.

Why this matters in AI
Filters are what CNNs learn. Every feature a CNN uses — edges, textures, object parts — is encoded in filter weights. Understanding filters explains why CNNs work and how to diagnose what a model is (or is not) detecting.
Common mistakes
  • Using very large filters (7×7 or 11×11) when stacking two 3×3 is more efficient — same receptive field with fewer parameters and two nonlinear activations.
  • Not using padding for intermediate conv layers — feature maps shrink to 1×1 after a few layers in deeper networks, losing spatial information.
  • Confusing filter (kernel) size with number of filters — kernel_size=3 is the spatial extent; out_channels=64 is how many distinct feature detectors you learn.

Pooling

Detailed explanation

Pooling layers reduce the spatial dimensions of feature maps by summarizing a small region with a single value. Max pooling takes the maximum value in each non-overlapping patch (typically 2×2 with stride=2), halving both height and width. Average pooling takes the mean. Global average pooling (GAP) reduces the entire feature map to a single value per channel — if you have a (64, 7, 7) feature map, GAP gives a (64,) vector.

Max pooling is the most common choice because it retains the strongest activation in each region — if a filter detected a strong edge anywhere in the 2×2 patch, that detection is preserved. This provides a small degree of translational invariance: a feature detected one pixel to the right or left still produces the same pooled value.

Pooling reduces computation and memory quadratically — halving spatial dimensions reduces computation by 4×. It also increases the effective receptive field of subsequent layers: after one max pool layer, each subsequent layer's 3×3 filter sees a 6×6 region of the original input. Stacking several rounds of conv + pool builds increasingly abstract representations over larger and larger image regions.

Global average pooling before the classifier has become the modern standard, replacing large fully connected layers. Instead of flattening a (512, 7, 7) feature map to 25,088 values and connecting to a Dense(25088, 1024) layer with 25 million parameters, GAP reduces (512, 7, 7) to (512,) and a single Dense(512, num_classes) with 512K parameters — a 50× reduction for a small accuracy loss. This also makes the model handle variable input sizes.

Strided convolutions are an alternative to pooling for downsampling. Instead of a separate pooling layer, use stride=2 in a conv layer. Many modern architectures (ResNet) use strided convolutions rather than max pooling for efficiency and because the downsampling is learned rather than fixed. Pooling remains common in simpler architectures and when computational efficiency is critical.

Input feature map (single channel, 4×4):
  [[1, 3, 2, 4],
   [5, 6, 7, 8],

Why this matters in AI
Pooling is the mechanism that builds spatial hierarchy in CNNs. Without it, deep networks would have insufficient receptive fields to recognize whole objects, only local textures.
Common mistakes
  • Applying max pooling with stride=1 — this downsamples nothing (the output is the same spatial size minus the kernel overlap). You need stride=2 to halve dimensions.
  • Using GAP when the spatial feature maps are not semantically meaningful — if early layers before classification still contain spatial noise, averaging loses the signal.
  • Pooling after the final conv block before a dense layer without flattening — the pooled tensor still has 3 spatial dimensions; flatten or use GAP before passing to nn.Linear.

Transfer learning

Detailed explanation

Transfer learning uses a model pre-trained on a large dataset (typically ImageNet, with 1.2 million images across 1000 classes) as a starting point for a new task. Instead of training a deep CNN from random weights on your small dataset, you start with weights that already encode rich visual features — edges, textures, shapes, object parts — and adapt them to your specific task.

There are two main approaches. Feature extraction freezes all pre-trained layers and only trains a new classification head on top. This is fast (few trainable parameters) and works well when your images are similar to ImageNet. Fine-tuning unfreezes some or all pre-trained layers and trains the entire model end-to-end on your data with a small learning rate. This adapts the pre-trained features to your domain and achieves higher accuracy but requires more data and care to avoid catastrophic forgetting.

The learning rate for fine-tuning must be small (1e-5 to 1e-4) to avoid destroying the pre-trained features. A common strategy is differential learning rates: very small lr for early layers (1e-5), slightly larger for middle layers (5e-5), and normal lr for the new classification head (1e-4). This preserves general early features while adapting later features to the new domain.

In PyTorch, torchvision.models provides pre-trained models: resnet50, efficientnet_b0, vit_b_16 etc. To use them: model = resnet50(weights='IMAGENET1K_V2'). To freeze all layers: for param in model.parameters(): param.requires_grad = False. To replace the classifier: model.fc = nn.Linear(model.fc.in_features, num_classes) — this new layer has requires_grad=True by default.

Domain gap determines how much transfer helps. ImageNet features transfer excellently to photos of animals, objects, and scenes. They transfer reasonably to medical imaging (textures and edges still matter). They transfer poorly to satellite imagery, microscopy, or highly specialized domains — where fine-tuning on domain data is essential. The worse the domain match, the more data you need and the more layers you should fine-tune.

Why this matters in AI
Transfer learning makes computer vision accessible with small datasets. A dataset of 500 images per class that would fail to train a CNN from scratch can achieve 90%+ accuracy with fine-tuned ResNet.
Common mistakes
  • Using a learning rate too high for fine-tuning (e.g., 1e-3 for pre-trained layers) — catastrophic forgetting rapidly overwrites pre-trained features, performing no better than training from scratch.
  • Freezing BatchNorm layers in train() mode — frozen BN still updates running stats. Use model.eval() for frozen BN layers to fix running stats.
  • Replacing the classifier head but forgetting to adjust input preprocessing — pre-trained ImageNet models expect ImageNet normalization (mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225]).

Image augmentation

Detailed explanation

Image augmentation artificially increases the diversity of the training set by applying random transformations to each training image every epoch. The model sees a different version of each image on every pass through the data, making it harder to memorize specific training examples and forcing generalization. Augmentation is one of the most effective techniques for improving model performance when data is limited.

Common augmentations for natural images: random horizontal flip (do not use vertical flip for upright photos of people or animals), random crop from a larger resize (e.g., resize to 256 then randomly crop 224×224), color jitter (randomly vary brightness, contrast, saturation, hue within a range), random rotation (±10–30 degrees), and random erasing (blank out a random rectangular region to force the model to use context).

The strength of augmentation should match your dataset. A dataset of 1000 images benefits from aggressive augmentation. A dataset of 100,000 images may not need much — adding too much noise hurts accuracy. Always validate: train with and without augmentation on the same validation set and check whether augmentation improves validation accuracy.

Domain-specific augmentation matters. Medical imaging uses elastic deformations (simulating tissue flexibility), Gaussian noise, and carefully calibrated rotations. Satellite imagery uses full 360-degree rotation and larger scale variation. Text-based augmentation uses synonym replacement and back-translation. The right augmentation strategy requires understanding what variations exist naturally in your data.

Albumentations is a powerful Python library for fast image augmentation that supports bounding boxes and segmentation masks — it applies the same spatial transform to both image and annotation. torchvision.transforms works for classification. For object detection and segmentation, use Albumentations or torchvision.transforms.v2.

Object detection

Detailed explanation

Object detection locates and classifies multiple objects in an image. Unlike classification (what is in this image?), detection answers: what objects are present, where exactly are they (bounding boxes), and what class is each? A bounding box is defined by 4 numbers: x_min, y_min, x_max, y_max or center_x, center_y, width, height.

Modern object detectors fall into two families. Two-stage detectors (Faster R-CNN) first propose candidate regions using a Region Proposal Network, then classify and refine each proposal. They are accurate but slower. One-stage detectors (YOLO, SSD, DETR) predict boxes and classes in a single forward pass, making them much faster and suitable for real-time applications like autonomous driving cameras.

YOLO (You Only Look Once) divides the image into a grid. Each grid cell predicts N bounding boxes and their confidence (is there an object here?) along with class probabilities. Post-processing uses Non-Maximum Suppression (NMS) to remove duplicate boxes: sort by confidence, keep the highest-confidence box, suppress all other boxes that overlap it by more than a threshold (typically IoU > 0.5).

Evaluation uses mean Average Precision (mAP). For each class, precision-recall is computed across different confidence thresholds. The area under the precision-recall curve is Average Precision (AP). mAP averages AP across all classes. IoU (Intersection over Union) threshold determines what counts as a correct detection — COCO standard uses mAP@[0.5:0.95] averaging over multiple IoU thresholds.

Training a detector from scratch requires large annotated datasets (COCO has 80 classes, 330,000 images) and significant compute. In practice, use a pre-trained detector (YOLOv8, Faster R-CNN pretrained on COCO) and fine-tune on your custom dataset using tools like Ultralytics YOLO or Detectron2.

Object detection is the core task in autonomous vehicles, surveillance, retail checkout, and medical imaging. Understanding the two-stage vs one-stage tradeoff helps you choose the right model for speed vs accuracy requirements.
Common mistakes

  • Not applying NMS post-processing — the model outputs many overlapping boxes per object; without NMS you get hundreds of boxes for one person.
  • Annotating bounding boxes inconsistently (including too much background, different labelers using different tightness) — inconsistent annotations are the main source of poor detector performance.

Segmentation

Detailed explanation

Image segmentation assigns a label to every pixel in an image, not just the whole image (classification) or a bounding box (detection). There are three levels: semantic segmentation labels every pixel with a class (all pixels belonging to a car get label 'car'), instance segmentation distinguishes different instances of the same class (car_1 pixels vs car_2 pixels), and panoptic segmentation combines both.

The standard semantic segmentation architecture is encoder-decoder. An encoder (like ResNet) progressively reduces spatial resolution while increasing feature depth. A decoder progressively upsamples back to the original resolution, recovering spatial detail. Skip connections between encoder and decoder layers (as in U-Net) help the decoder access fine-grained spatial information that was lost during downsampling.

U-Net was designed for medical image segmentation and has become the dominant architecture across many domains. It takes an image, encodes it through a series of conv + pool blocks (contracting path), then decodes through a series of upsampling + conv blocks (expanding path), with skip connections concatenating encoder features to the decoder at each resolution. The output is a per-pixel class map the same size as the input.

Evaluation uses mean Intersection over Union (mIoU). For each class, IoU = true positives / (true positives + false positives + false negatives) at the pixel level. mIoU averages across all classes. A model with mIoU of 0.75 on autonomous driving data means on average 75% of pixels for each class are correctly predicted.

Instance segmentation (Mask R-CNN) extends Faster R-CNN by adding a mask prediction head: for each detected object box, a small FCN predicts a 28×28 binary mask of which pixels inside the box belong to the object. Segment Anything Model (SAM) from Meta can segment any object in any image given a prompt (point, box, or text), making high-quality segmentation accessible without task-specific training.

Loss: CrossEntropyLoss per pixel. For class imbalance (e.g., tiny lesion in large background), use weighted loss or Dice loss.
Why this matters in AI
Segmentation is the foundation of medical AI (tumor delineation), autonomous driving (lane and pedestrian detection), and satellite analysis (building footprint extraction). Pixel-level accuracy is required where bounding boxes are insufficient.
Common mistakes
  • Using accuracy as the metric for segmentation with class imbalance — if background is 95% of pixels, predicting all background achieves 95% accuracy while being useless for detecting the rare class.
  • Not using skip connections in encoder-decoder — without them, the decoder cannot recover fine spatial detail and produces blurry segmentation boundaries.
  • Training with randomly cropped patches without ensuring each patch contains some foreground — the model may only ever see background and never learn to detect the class of interest.

OCR

Detailed explanation

OCR (Optical Character Recognition) converts images containing text into machine-readable text. Modern OCR combines deep learning for text detection (locating where text is in the image) and text recognition (reading the text at each location). OCR is a prerequisite for Document AI: processing scanned documents, PDFs, receipts, and forms.

Tesseract: the most widely used open-source OCR engine (maintained by Google). Works well for high-quality scanned text with clear fonts. pytesseract provides a Python wrapper. Best performance requires preprocessing: convert to grayscale, apply adaptive thresholding, deskew (correct rotation). Struggles with: handwritten text, complex layouts, low-resolution images, tables.

EasyOCR: a Python library supporting 80+ languages. Better than Tesseract on many real-world documents. Uses CRAFT (text detection) + CRNN (text recognition). `reader = easyocr.Reader(['en', 'fr']); results = reader.readtext(image_path)` returns a list of (bounding_box, text, confidence) tuples.

PaddleOCR: the state-of-the-art open-source OCR from Baidu. PP-OCR v4 is extremely accurate on complex document layouts, mixed languages, and rotated text. Pipeline: text detection (DBNet) → text direction classification → text recognition (SVTR). Significantly outperforms Tesseract and EasyOCR on challenging documents.

Cloud OCR: Google Cloud Vision API, AWS Textract, and Azure Computer Vision provide managed OCR. Textract additionally extracts structured data from forms and tables (key-value pairs, table cells) with high accuracy on standard business documents. Use cloud OCR for production when: high accuracy is required, documents have complex layouts, or you need structured extraction (forms, invoices) with minimal custom training.

import easyocr
from PIL import Image
import numpy as np
# EasyOCR - easiest to get started
reader = easyocr.Reader(['en'], gpu=False)  # gpu=True if CUDA available
results = reader.readtext('invoice.jpg')
for (bbox, text, confidence) in results:
    print(f'Text: {text:30s} | Confidence: {confidence:.3f}')
    # bbox = [[x1,y1], [x2,y2], [x3,y3], [x4,y4]] (four corners)
# Extract all text as a string
full_text = ' '.join([text for (_, text, _) in results])
print('Full document text:', full_text[:200])
# Tesseract with preprocessing
import pytesseract
import cv2
img = cv2.imread('document.png')
# Preprocessing for better OCR accuracy
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
# Adaptive thresholding (handles uneven lighting)
thresh = cv2.adaptiveThreshold(gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C, cv2.THRESH_BINARY, 11, 2)
# Deskew (straighten tilted scans)
def deskew(image):
    coords = np.column_stack(np.where(image > 0))
    angle = cv2.minAreaRect(coords)[-1]
    if angle < -45: angle = 90 + angle
    (h, w) = image.shape[:2]
    M = cv2.getRotationMatrix2D((w//2, h//2), -angle, 1.0)
    return cv2.warpAffine(image, M, (w, h), flags=cv2.INTER_CUBIC)
deskewed = deskew(thresh)
# OCR
text = pytesseract.image_to_string(deskewed, config='--psm 6')  # psm 6: uniform block of text
print(text)
Why this matters in AI
OCR is the entry point for processing the vast majority of business documents — contracts, invoices, medical records, and reports exist as scanned PDFs or images. A Document AI pipeline that can OCR any document and extract structured information from it can automate enormous amounts of manual data entry. Understanding the OCR pipeline from preprocessing to structured extraction is a high-value practical skill.
Common mistakes
  • Not preprocessing images before OCR — contrast enhancement, thresholding, and deskewing improve OCR accuracy significantly on real-world documents. Don't send a raw low-contrast scan to OCR and expect good results.
  • Using Tesseract on complex layouts — Tesseract assumes a uniform text layout. For documents with columns, tables, and mixed text+graphics, use PaddleOCR or a cloud OCR service that handles complex layouts.
  • Not validating OCR output — OCR always makes some errors. For production pipelines, validate extracted values against expected formats (dates, amounts, IDs) and flag low-confidence extractions for human review.

Vision Transformers

Detailed explanation

Vision Transformers (ViT) apply the transformer architecture (designed for sequences of tokens in NLP) directly to images by splitting the image into fixed-size non-overlapping patches, flattening each patch into a vector, and treating the sequence of patch vectors as a sequence of tokens — just like words in a sentence. A 224×224 image divided into 16×16 patches produces (224/16)^2 = 196 patches, each of size 16×16×3 = 768 dimensions.

The patch embeddings are linearly projected to the model dimension d, a learnable [CLS] token is prepended (whose final representation is used for classification), and positional embeddings are added to give the model information about patch location. This sequence of 197 vectors (196 patches + 1 CLS) is fed through a standard transformer encoder with multi-head self-attention and MLP layers.

ViT requires much more training data than CNNs to perform well from scratch. CNNs have inductive biases baked in — locality (nearby pixels relate) and translational equivariance (a filter works anywhere in the image). ViTs have no such inductive bias and must learn these properties from data. On ImageNet alone (1.2M images), a ViT underperforms ResNet. Pre-trained on 300M images (JFT-300M), ViT-Large surpasses ResNet. DeiT (Data-efficient Image Transformers) uses knowledge distillation to train competitive ViTs on ImageNet alone.

Self-attention in ViT is global: every patch attends to every other patch in one layer. This allows long-range dependencies (a patch at the top of the image can directly attend to one at the bottom) that CNNs can only develop through many stacked layers. This is why ViTs excel at tasks requiring global context, like recognizing relationships between distant parts of an image.

Modern vision models often combine both: Swin Transformer uses a hierarchical structure with local window attention (like CNN locality) and shifted windows for cross-window communication, combining CNN efficiency with transformer global reasoning. It has become the backbone of choice for detection and segmentation in 2022–2024.

Input: [B, 3, 224, 224]. Patch embed: [B, 196, 768]. Add [CLS] token: [B, 197, 768].
After transformer encoder (12 layers, 12 heads): [B, 197, 768]. Take CLS: [B, 768].
MLP head: Linear(768, 10). Output: [B, 10] logits.
Fine-tune: Adam with lr=1e-4 for head, 1e-5 for backbone. ViTs need smaller lr than CNNs.
Why this matters in AI
ViTs are now state-of-the-art for image classification, generation (DiT), and video understanding. Understanding their patch-based input and global attention explains their strengths and why they need more data than CNNs.
Common mistakes
  • Training ViT from scratch on small datasets (<100K images) — ViT will underfit significantly without large-scale pre-training. Always use a pre-trained ViT.
  • Using the same learning rate for ViT as for ResNet — ViTs are more sensitive to learning rate; use a warmup schedule and smaller lr.
  • Forgetting positional embeddings — without them, the model cannot distinguish which patch came from which position in the image.
