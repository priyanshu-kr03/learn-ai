Probability

Detailed explanation

Probability is the mathematical language for reasoning under uncertainty. The probability of an event E is a number P(E) between 0 and 1: P(E)=0 means the event never occurs; P(E)=1 means it always occurs. For a fair die, P(rolling a 6) = 1/6 ≈ 0.167. Probability theory is the mathematical backbone of machine learning: every ML model outputs probabilities (explicitly or implicitly), and training is optimization over probability distributions.

Joint probability P(A and B) is the probability that both A and B occur. Conditional probability P(A|B) is the probability of A given that B has occurred. Independence: A and B are independent if P(A|B) = P(A) — knowing B gives no information about A. For independent events: P(A and B) = P(A) * P(B). The chain rule: P(A and B) = P(A|B) * P(B) — any joint probability can be decomposed into conditionals.

Probability distributions describe how probability is spread across outcomes. Discrete distributions assign probabilities to countable outcomes (Bernoulli: one binary event; Categorical: one of K classes; Binomial: number of successes in N trials). Continuous distributions assign probability density to continuous outcomes (Normal: bell curve; Exponential: time between events; Beta: probability of a probability). ML models predict parameters of these distributions.

Expectation E[X] is the probability-weighted average of all possible values: E[X] = Σ x * P(X=x) for discrete distributions. For a fair die, E[roll] = 1*(1/6) + 2*(1/6) + ... + 6*(1/6) = 3.5. In ML, the expected loss E[L(y, ŷ)] is what training minimizes. The empirical expectation (mean over training data) approximates the true expectation.

The Law of Large Numbers states that the sample mean converges to the true expectation as n → ∞. This justifies using training data to estimate model parameters — the more data, the closer your empirical estimates are to the true distribution. The Central Limit Theorem states that the sample mean of any distribution (with finite variance) converges to a normal distribution as n → ∞, enabling statistical testing.

Why this matters in AI
Probability is the language of ML. Loss functions (cross-entropy, KL divergence) are defined in terms of probability. Model outputs are probabilities. Training is maximum likelihood estimation. Bayesian methods quantify uncertainty via probability distributions. Without probability, ML is just magic.
Common mistakes
  • Confusing P(A|B) with P(B|A) — this is the base rate fallacy. P(positive test | disease) ≠ P(disease | positive test).
  • Multiplying probabilities for dependent events — P(A and B) = P(A)*P(B) is only valid if A and B are independent. For dependent events, use P(A and B) = P(A|B)*P(B).
  • Assuming uniform probability when it isn't — not all outcomes are equally likely. Always check the prior distribution of the problem.

Random variables

Detailed explanation

A random variable is a variable whose value is determined by the outcome of a random experiment. It maps outcomes of a random process to numbers. For a coin flip: X = 1 for heads, X = 0 for tails. For a die roll: X ∈ {1, 2, 3, 4, 5, 6}. For stock price change: X ∈ R (any real number). Random variables formalize the notion of 'a quantity that is uncertain'.

Discrete random variables take countable values (integers, specific categories). Probability mass function (PMF) P(X=x) gives the probability of each value. Σ P(X=x) = 1 (probabilities sum to 1). Continuous random variables take uncountable values (real numbers). Probability density function (PDF) f(x): P(a ≤ X ≤ b) = ∫f(x)dx from a to b. The PDF gives density, not probability — P(X = x) = 0 for any exact value.

The Cumulative Distribution Function (CDF) F(x) = P(X ≤ x) is defined for both discrete and continuous RVs. It gives the probability that X takes a value ≤ x. The CDF is monotonically non-decreasing, from 0 to 1. For a continuous RV, the PDF is the derivative of the CDF: f(x) = dF/dx. sklearn's empirical CDF can be approximated with np.percentile.

Transformations of random variables: if X ~ N(0,1), then Y = μ + σX ~ N(μ, σ²). If X ~ N(μ, σ²), then Z = (X-μ)/σ ~ N(0,1). If X is a random variable with PDF f(x), then Y = g(X) is also a random variable. Understanding transformations enables creating any distribution from a standard uniform [0,1] via the inverse CDF method.

In ML, model outputs are random variables. A logistic regression outputs P(y=1|x) — the probability of the positive class. This is a Bernoulli random variable conditioned on x. A Gaussian process outputs a distribution (mean and variance) over function values. VAEs encode inputs as distributions (means and variances) in latent space. Understanding random variables enables you to understand probabilistic models.

Why this matters in AI
Random variables are the mathematical language of probability and statistics. Every ML model makes probabilistic assumptions about data generation. Understanding random variables enables you to understand loss functions (cross-entropy = negative log-likelihood of a Bernoulli RV), regularization (priors over weights), and model uncertainty.
Common mistakes
  • Confusing the PDF value with probability — f(x) for a continuous RV is density, not probability. P(X=5) = 0 for a continuous RV. Probability is the integral of the PDF over an interval.
  • Confusing the PMF of a discrete RV with the PDF — for a discrete RV, probabilities sum to 1; for a continuous RV, the PDF integrates to 1 but individual PDF values can exceed 1.
  • Ignoring independence assumptions — many ML methods assume feature values are independent random variables. When features are correlated, these assumptions are violated.

Conditional probability

Detailed explanation

Conditional probability P(A | B) is the probability of event A occurring given that event B has occurred. Formula: P(A | B) = P(A ∩ B) / P(B) — the probability of both A and B happening, divided by the probability of B. Example: P(disease | positive_test) is the probability of actually having the disease given a positive test — this can be very different from P(positive_test | disease) (the test sensitivity).

The distinction between P(A|B) and P(B|A) is critical and counterintuitive. Medical example: P(positive_test | disease) = 0.99 (test is 99% sensitive). P(disease | positive_test) depends on the disease prevalence (base rate). If only 1 in 10,000 people have the disease: P(disease | positive_test) ≈ 1% even with a 99% sensitive test. This is Bayes theorem in action and explains why rare disease screening is difficult.

Bayes theorem: P(A | B) = P(B | A) × P(A) / P(B). In ML: P(class | features) = P(features | class) × P(class) / P(features). The prior P(class) is the class frequency. The likelihood P(features | class) is what Naive Bayes models. The posterior P(class | features) is what we want — the classification output. Bayes theorem is the mathematical foundation of probabilistic ML.

Joint probability: P(A ∩ B) = P(A | B) × P(B) = P(B | A) × P(A). Independence: A and B are independent if P(A | B) = P(A) — knowing B doesn't change the probability of A. P(A ∩ B) = P(A) × P(B) when independent. Naive Bayes assumes features are conditionally independent given the class: P(x₁, x₂ | class) = P(x₁ | class) × P(x₂ | class).

Conditional probability in ML: (1) Bayesian inference: update beliefs (prior) with evidence (likelihood) to get posterior probability. (2) Causal inference: P(Y | do(X)) ≠ P(Y | X) — interventional vs observational probability. (3) Language models: P(next_word | previous_words) is a conditional probability — GPT predicts the next token given all previous tokens. (4) Bayesian neural networks: instead of point estimates of weights, maintain a posterior distribution P(weights | data).

Worked example

# Bayes theorem example: disease screening
P_disease = 0.0001      # 1 in 10,000 have the disease
P_pos_given_disease = 0.99  # test sensitivity
P_pos_given_healthy = 0.01  # 1% false positive rate
P_pos = P_pos_given_disease * P_disease + P_pos_given_healthy * (1 - P_disease)
P_disease_given_pos = P_pos_given_disease * P_disease / P_pos
print(f'P(disease | positive test) = {P_disease_given_pos:.4f}')  # ~0.98% !
# Conditional probability from data
import pandas as pd
# P(default | income < 30k)
mask = df['income'] < 30000
print(f'P(default | income<30k): {df.loc[mask, "default"].mean():.3f}')
print(f'P(default): {df["default"].mean():.3f}')
Why this matters in AI
Conditional probability is the mathematical language of ML. Every ML model computes P(output | input) — a conditional probability. Understanding conditional probability prevents the base rate neglect fallacy (ignoring class imbalance when interpreting model outputs) and explains why classification thresholds must account for class frequencies.
Common mistakes
  • Confusing P(A|B) with P(B|A) — these are completely different probabilities. P(test positive | cancer) ≠ P(cancer | test positive). The prosecutor's fallacy in legal contexts (P(evidence | innocent) ≠ P(innocent | evidence)) is this exact error.
  • Ignoring the prior (base rate) when computing posterior probability — Bayes theorem requires P(A) (the prior). Ignoring the low base rate of rare diseases makes tests seem more informative than they are.
  • Assuming independence without checking — P(A ∩ B) = P(A) × P(B) only when A and B are independent. In real data, features are almost always correlated.

Bayes theorem

Detailed explanation

Bayes' theorem describes how to update beliefs (probabilities) when new evidence is observed. It states: P(H|E) = P(E|H) * P(H) / P(E). In English: the probability of hypothesis H given evidence E equals the likelihood of seeing evidence E if H is true, times the prior probability of H, divided by the total probability of seeing E.

The four components: P(H|E) is the posterior — what we believe about H after seeing E. P(H) is the prior — what we believed about H before seeing E. P(E|H) is the likelihood — how probable is E if H is true. P(E) is the marginal likelihood (evidence) — how probable is E overall (normalizing constant ensuring the posterior is a valid probability).

Medical testing example: a disease affects 1% of the population (prior P(disease)=0.01). A test has 90% sensitivity (P(positive|disease)=0.9) and 95% specificity (P(negative|no disease)=0.95, so P(positive|no disease)=0.05). You test positive. What is the probability you have the disease? P(disease|positive) = 0.9 * 0.01 / (0.9*0.01 + 0.05*0.99) = 0.009 / (0.009 + 0.0495) = 0.154 ≈ 15%. Despite a 90% sensitive test, a positive result is only 15% likely to indicate disease because the disease is rare.

Bayesian inference extends this to continuous parameter estimation. In Bayesian linear regression, instead of a single weight estimate w, you maintain a distribution over weights P(w|data). This distribution captures uncertainty: if data is limited, the posterior is wide (uncertain weights); with more data, it narrows. Bayesian methods naturally handle uncertainty and prevent overconfident point estimates.

Naive Bayes classifier applies Bayes theorem to classification with the 'naive' assumption that features are conditionally independent given the class. Despite this unrealistic assumption, Naive Bayes works surprisingly well for text classification (spam detection) and is extremely fast to train. It is still a strong baseline for NLP tasks.

Common mistakes

Bernoulli distribution

Detailed explanation

The Bernoulli distribution models a single trial with two outcomes: success (1) with probability p and failure (0) with probability 1-p. It is the simplest discrete probability distribution and the building block for more complex distributions. ML examples: a single email being spam (p=0.1 chance of spam), a user clicking an ad, a patient having a disease, a coin flip.

PMF (Probability Mass Function): P(X=1) = p, P(X=0) = 1-p. Mean = p. Variance = p(1-p). The variance is maximized at p=0.5 (maximum uncertainty) and decreases as p approaches 0 or 1. This formula appears in the information-theoretic entropy of a binary variable: H(p) = -p log(p) - (1-p) log(1-p).

ML connection to logistic regression: logistic regression models P(Y=1|X) = σ(w·x + b) where σ is the sigmoid function. The output is a Bernoulli probability. The loss function (binary cross-entropy) is exactly the negative log-likelihood of the Bernoulli distribution. Training logistic regression = finding weights that maximize the probability that each training label was drawn from a Bernoulli with the predicted probability.

Maximum likelihood estimation: given n observations of a Bernoulli random variable, the MLE of p is simply the sample proportion: p̂ = (number of 1s) / n. This is intuitively obvious but important: the ML estimate of the Bernoulli parameter is the fraction of successes. This generalizes to categorical distributions (the multinomial MLE is the fraction of each outcome).

Connection to other distributions: Binomial(n, p) = sum of n independent Bernoulli(p) variables. If you flip a coin 100 times (100 Bernoulli trials), the total number of heads follows Binomial(100, 0.5). This connection means understanding Bernoulli is the gateway to understanding Binomial, and by extension, the basis of many statistical tests.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
from scipy import stats
import matplotlib.pyplot as plt
# Bernoulli distribution
p = 0.3  # probability of success
dist = stats.bernoulli(p)
print(f'P(X=1) = {dist.pmf(1):.2f}')  # 0.30
print(f'P(X=0) = {dist.pmf(0):.2f}')  # 0.70
print(f'Mean = {dist.mean():.2f}')     # 0.30
print(f'Variance = {dist.var():.4f}')  # 0.21 = p(1-p)
# Simulate Bernoulli trials
np.random.seed(42)
samples = dist.rvs(size=1000)  # 1000 Bernoulli trials
print(f'Simulated fraction of 1s: {samples.mean():.3f}')  # should be ≈ 0.3
# MLE of p from observations
def mle_bernoulli(observations):
    return observations.mean()  # simplest MLE
print(f'MLE estimate of p: {mle_bernoulli(samples):.3f}')
Why this matters in AI
The Bernoulli distribution is the mathematical foundation of binary classification. When you train a logistic regression or a neural network for binary classification, you are implicitly assuming the labels follow a Bernoulli distribution. The binary cross-entropy loss IS the negative log-likelihood of the Bernoulli distribution. Understanding this connection lets you reason clearly about why models fail on imbalanced classes (low p means most labels are 0, so predicting all-zeros is a strong baseline).
Common mistakes
  • Confusing Bernoulli(p) and Binomial(1, p) — they are equivalent, but Bernoulli refers to a single trial while Binomial(1, p) is often the parameterization in software. When using scipy.stats.binom(n=1, p=0.3) vs bernoulli(0.3), the results are identical.
  • Using binary cross-entropy loss for class-imbalanced data without reweighting — when p is very small (1% positive class), the model that predicts 0 always has 99% accuracy but 0% recall. Add class weights or use focal loss to address this.
  • Not understanding the variance formula — p(1-p) shows that uncertainty is highest at p=0.5. When building classifiers for rare events (p=0.01), predictions near p=0.01 have very low variance, meaning small changes in the model produce large changes in relative precision/recall.

Binomial distribution

Detailed explanation

The Binomial distribution models the number of successes in n independent Bernoulli trials, each with success probability p. If you flip a biased coin (P(heads)=0.3) ten times, the number of heads follows Binomial(n=10, p=0.3). ML examples: number of correctly classified images out of 100, number of customers who convert out of 1,000 shown an ad, number of fraudulent transactions in a batch of 500.

PMF: P(X=k) = C(n,k) × p^k × (1-p)^(n-k). Mean = n×p. Variance = n×p×(1-p). Standard deviation = sqrt(n×p×(1-p)). The binomial coefficient C(n,k) = n!/(k!(n-k)!) counts the number of ways to choose k successes out of n trials. For large n, the Binomial approximates the Normal distribution by the Central Limit Theorem.

Statistical testing with the Binomial: is a model's accuracy of 60% on 100 test examples significantly better than random (50%)? Use a binomial test: H0: p=0.5, observed k=60 successes out of n=100 trials. The p-value is P(X >= 60 | p=0.5). If this is small (< 0.05), the result is statistically significant. `scipy.stats.binom_test(60, 100, 0.5)` computes this directly.

A/B testing connection: click-through rates are Binomial variables. Variant A: 1200 clicks out of 10,000 impressions (p=0.12). Variant B: 1350 clicks out of 10,000 impressions (p=0.135). Is the difference significant? Chi-squared test or two-proportion z-test. Understanding the Binomial is foundational for designing statistically valid A/B tests.

Normal approximation: for large n (n×p > 5 and n×(1-p) > 5), Binomial(n, p) ≈ Normal(μ=n×p, σ²=n×p×(1-p)). This approximation is why confidence intervals for proportions are based on the normal distribution: 95% CI for p = p̂ ± 1.96 × sqrt(p̂×(1-p̂)/n).

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

from scipy import stats
import numpy as np
import matplotlib.pyplot as plt
# Binomial distribution
n, p = 20, 0.3
dist = stats.binom(n, p)
print(f'Mean: {dist.mean():.1f} (expected: {n*p})')
print(f'Variance: {dist.var():.2f} (expected: {n*p*(1-p):.2f})')
# PMF: probability of exactly k successes
for k in range(0, n+1):
    if dist.pmf(k) > 0.05:
        print(f'P(X={k}) = {dist.pmf(k):.4f}')
# CDF: probability of k or fewer successes
print(f'P(X <= 6) = {dist.cdf(6):.4f}')
# Confidence interval for observed proportion
k_observed = 60; n_trials = 100
ci = stats.proportion_confint(k_observed, n_trials, alpha=0.05, method='wilson')
print(f'95% CI for p=0.60: [{ci[0]:.3f}, {ci[1]:.3f}]')
# Statistical test: is 60/100 better than 50%?
pval = stats.binom_test(60, 100, p=0.5, alternative='greater')
print(f'p-value: {pval:.4f}')  # 0.028 — significant at 5%
Why this matters in AI
Binomial distributions appear everywhere in ML evaluation. Every metric computed on a finite test set is an estimate with a margin of error. A model with 92% accuracy on 50 test examples has a wide confidence interval — it could be anywhere from 80% to 98%. A model with 92% accuracy on 5000 test examples has a much tighter interval (90.5%-93.5%). The Binomial distribution quantifies this uncertainty.
Common mistakes
  • Reporting accuracy on small test sets without confidence intervals — 10/10 correct sounds like perfect accuracy but the 95% CI includes everything from 69% to 100%. Always report confidence intervals with performance metrics.
  • Using the normal approximation when n×p < 5 — for rare events (p=0.01, n=100, n×p=1), the normal approximation fails badly. Use the exact Binomial (scipy.stats.binom_test) for rare events.
  • Performing multiple statistical tests without correction — if you compare 20 metrics and one shows p<0.05, the expected number of false positives is 1 (by chance alone). Use Bonferroni correction or FDR control when testing multiple hypotheses.

Normal distribution

Detailed explanation

The normal distribution (also called the Gaussian distribution) is the most important probability distribution in statistics and machine learning. It is bell-shaped, symmetric around its mean μ, and characterized by two parameters: mean μ (center of the bell) and standard deviation σ (width of the bell). The probability density function is: f(x) = (1/σ√2π) * exp(-(x-μ)²/2σ²).

The 68-95-99.7 rule: 68% of data falls within 1σ of the mean, 95% within 2σ, 99.7% within 3σ. A data point 3 standard deviations from the mean is very rare (0.3% probability). This rule lets you quickly identify outliers: a value more than 3σ from the mean warrants investigation in most datasets.

Why is the normal distribution everywhere? The Central Limit Theorem: the sum (or average) of many independent random variables converges to a normal distribution regardless of their individual distributions, as long as they have finite mean and variance. This explains why measurement errors, heights, weights, IQ scores, and many other naturally occurring quantities follow approximately normal distributions.

In ML, the normal distribution appears as: the weight initialization in neural networks (He/Xavier initialization draws from Gaussian), the noise model in linear regression (the error terms are assumed Gaussian), the likelihood in Gaussian Naive Bayes, the basis of Gaussian Processes, and the distribution parameterized by VAE latent spaces. Understanding it is prerequisite to understanding many model assumptions.

Standard normal distribution: when μ=0 and σ=1, it is called the standard normal distribution, denoted Z ~ N(0,1). Any normal distribution can be standardized: Z = (X - μ)/σ. Z-scores tell you how many standard deviations a value is from the mean. sklearn's StandardScaler computes z-scores for each feature.

scipy CDF: from scipy.stats import norm; p = norm.cdf(180, mu, sigma)  # P(height <= 180cm)
Why this matters in AI
The normal distribution is the mathematical foundation of many ML algorithms, statistical tests, and uncertainty quantification methods. Understanding it deeply enables you to recognize when data violates normality assumptions and what consequences that has.
Common mistakes
  • Assuming all real data is normally distributed — income, transaction amounts, and many other real-world quantities are highly skewed (log-normal or power-law). Test normality before assuming it.
  • Confusing mean and median for skewed distributions — for a normal distribution, mean = median. For skewed data, they diverge significantly.
  • Using z-score outlier detection on non-normal data — z-scores assume normality; on skewed data, real outliers on one side are missed and non-outliers on the other are flagged.

Poisson distribution

Detailed explanation

The Poisson distribution models the number of events that occur in a fixed interval of time or space, given that these events happen independently at a constant average rate λ (lambda). ML examples: number of support tickets per hour, number of transactions per minute, number of fraudulent charges per day, number of user clicks per session.

PMF: P(X=k) = (λ^k × e^(-λ)) / k! where k=0,1,2,3,... Mean = λ. Variance = λ (the mean equals the variance — this is the defining property of the Poisson distribution). For large λ, the Poisson approximates Normal(λ, sqrt(λ)).

Poisson regression: used when the outcome is a count variable (number of events). Unlike linear regression (which can predict negative values), Poisson regression models log(E[Y]) = β₀ + β₁X₁ + ... which ensures predictions are always non-negative. GLM with Poisson family. Sklearn doesn't have Poisson regression directly — use statsmodels or sklearn's HistGradientBoostingRegressor with Poisson loss.

Overdispersion: real count data often has variance > mean (overdispersion), violating the Poisson assumption. Social media posts per user: a few power users skew variance up. Fix: use Negative Binomial regression (generalizes Poisson to allow variance > mean). In sklearn: HistGradientBoostingRegressor with loss='poisson' handles overdispersion better than classical Poisson regression.

Checking if data is Poisson: plot the observed distribution vs Poisson(λ=mean). If they match, Poisson is appropriate. If the observed has a heavier tail (more high-count events than Poisson predicts), the data is overdispersed — use Negative Binomial. If the observed has more zeros than Poisson predicts, consider Zero-Inflated Poisson (ZIP).

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
from scipy import stats
import matplotlib.pyplot as plt
# Poisson distribution
lambda_val = 3.5  # average events per interval
dist = stats.poisson(lambda_val)
k_values = np.arange(0, 15)
print('P(X=k) for Poisson(λ=3.5):')
for k in range(8):
    print(f'P(X={k}) = {dist.pmf(k):.4f}')
print(f'Mean: {dist.mean()}, Variance: {dist.var()}')  # both = λ
# Compare actual data to Poisson
observed_counts = np.random.poisson(lambda_val, size=1000)
observed_distribution = np.bincount(observed_counts, minlength=15)
expected_distribution = dist.pmf(k_values) * 1000
fig, ax = plt.subplots(figsize=(10, 4))
ax.bar(k_values - 0.2, observed_distribution, width=0.4, label='Observed', alpha=0.7)
ax.bar(k_values + 0.2, expected_distribution, width=0.4, label='Poisson(3.5)', alpha=0.7)
ax.legend(); ax.set_xlabel('Count'); ax.set_ylabel('Frequency')
Why this matters in AI
The Poisson distribution is the correct model for count data (how many events happen?), and using linear regression for count data leads to nonsensical negative predictions and poor calibration. Recognizing count data and fitting Poisson models (or their generalizations) is a key statistical modeling skill for ML practitioners working with event data, transaction logs, and operational metrics.
Common mistakes
  • Using linear regression for count outcomes — predicting 'number of support tickets tomorrow' with linear regression can produce negative predictions. Poisson regression guarantees non-negative predictions by modeling log(E[Y]).
  • Assuming Poisson when data is overdispersed — if variance >> mean in your count data, the Poisson assumption is violated. Always check: mean and variance of count outcome. If variance > 2× mean, use Negative Binomial regression.
  • Not accounting for exposure — 'number of accidents per store' is unfair if stores have different sizes (exposure). Use an offset term: log(n_days) as an offset in Poisson regression converts 'raw count' to 'rate per day' automatically.

Expectation variance

Detailed explanation

Expectation (expected value) E[X] is the average value of a random variable X if you observe it infinitely many times. For a discrete random variable: E[X] = sum_x(x × P(X=x)). For a continuous random variable: E[X] = integral(x × f(x) dx) where f(x) is the probability density. Expectation gives the center of mass of the distribution.

Variance Var(X) = E[(X - E[X])^2] = E[X^2] - (E[X])^2 measures how spread out the distribution is — the average squared deviation from the mean. Standard deviation = sqrt(Var(X)) is the variance in the same units as X. A normal distribution with mean=0, std=1 has Var=1. High variance = distribution is spread out. Low variance = distribution is concentrated around the mean.

Properties: E[aX + b] = a×E[X] + b (linearity). Var(aX + b) = a^2 × Var(X) (constants shift expectation but adding a constant doesn't change spread; multiplying by a scales spread by a^2). E[X + Y] = E[X] + E[Y] (linearity of expectation, always true). Var(X + Y) = Var(X) + Var(Y) ONLY when X and Y are independent (otherwise: Var(X+Y) = Var(X) + Var(Y) + 2×Cov(X,Y)).

ML relevance: expected loss E_data[L(f(x), y)] is what we minimize in training. Sample mean is an unbiased estimator of E[X]: E[sample_mean] = E[X]. The bias-variance tradeoff decomposes expected prediction error as: E[(y - f(x))^2] = Bias^2 + Variance + Irreducible Noise. High bias = model underfits (predictions consistently wrong). High variance = model overfits (predictions vary wildly across different training sets).

Central Limit Theorem (CLT): for any distribution with finite mean μ and variance σ^2, the sample mean of n independent draws is approximately Normal(μ, σ^2/n). This is why: bootstrapped confidence intervals work, hypothesis tests based on the normal distribution are valid for large samples, and feature scaling is important (many normalized signals approximate normal distributions). The CLT is the most important result in statistics for ML practitioners.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
from scipy import stats
# Empirical vs theoretical expectation/variance
np.random.seed(42)
# Dice roll: X ~ Uniform{1,2,3,4,5,6}
samples = np.random.randint(1, 7, size=100000)
print(f'E[X] empirical: {samples.mean():.4f}, theoretical: {(1+6)/2:.4f}')  # 3.5
print(f'Var(X) empirical: {samples.var():.4f}, theoretical: {((6-1+1)**2-1)/12:.4f}')  # 35/12
# Bernoulli: E[X] = p, Var(X) = p(1-p)
p = 0.3
bern = np.random.binomial(1, p, size=10000)
print(f'Bernoulli E[X]: {bern.mean():.4f} (expected {p})')
print(f'Bernoulli Var(X): {bern.var():.4f} (expected {p*(1-p):.4f})')
# CLT demonstration
fig, axes = plt.subplots(1, 3, figsize=(12,4))
for i, n in enumerate([1, 10, 100]):
    sample_means = [np.random.exponential(1, n).mean() for _ in range(10000)]
    axes[i].hist(sample_means, bins=50, density=True)
    axes[i].set_title(f'n={n}: sample mean distribution')
    print(f'n={n}: std of sample mean = {np.std(sample_means):.4f} (expected {1/np.sqrt(n):.4f})')
Why this matters in AI
Expectation and variance are the language of probabilistic thinking in ML. When you compute training loss, you are estimating expected loss. When you worry about overfitting, you are concerned about high variance. When you run A/B tests, you use CLT to construct confidence intervals. Fluency with E[X] and Var(X) makes statistical reasoning automatic.
Common mistakes
  • Var(X+Y) = Var(X) + Var(Y) only for independent X and Y — if X and Y are correlated, Var(X+Y) = Var(X) + Var(Y) + 2×Cov(X,Y). This is why diversifying across correlated assets reduces portfolio variance less than diversifying across uncorrelated assets.
  • Confusing expected value with the most probable value — E[X] minimizes squared error predictions, while the mode minimizes 0/1 error. For a skewed distribution, E[X] ≠ mode(X). For a Bernoulli(p=0.1): E[X]=0.1 but the most probable value is 0.
  • Forgetting that sample variance is biased — np.var(x) divides by n (biased). np.var(x, ddof=1) divides by n-1 (Bessel's correction, unbiased). Use ddof=1 for sample variance to get an unbiased estimate of the population variance.
