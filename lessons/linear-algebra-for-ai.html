Scalars vectors matrices tensors

Detailed explanation

Scalar: a single number with no dimensions (rank 0 tensor). Examples: learning rate = 0.001, loss = 0.45, accuracy = 0.87. In NumPy: np.float32(0.001). In PyTorch: torch.tensor(0.001).

Vector: a 1D array of scalars (rank 1 tensor). Examples: a word embedding [0.1, -0.3, 0.7, ...] of dimension 512, a sample feature vector [age, income, score] = [25, 50000, 0.8], a probability distribution [0.7, 0.2, 0.1] over 3 classes. In NumPy: np.array([1.0, 2.0, 3.0]). Vectors have a magnitude (length) and direction in n-dimensional space.

Matrix: a 2D array of scalars (rank 2 tensor). Examples: a weight matrix W of shape (input_dim, output_dim), a dataset X of shape (n_samples, n_features), an attention matrix of shape (seq_len, seq_len). Matrix multiplication combines two matrices into a new matrix. The rows represent one dimension (samples, sequence positions), columns represent another (features, hidden dimensions).

Tensor: a generalization to any number of dimensions (rank ≥ 3). Examples: a batch of images (batch_size, channels, height, width) = (32, 3, 224, 224) — rank 4, a batch of text sequences (batch_size, seq_len, embedding_dim) = (32, 128, 768) — rank 3, a batch of videos (batch, frames, channels, height, width) — rank 5. PyTorch and TensorFlow are named after tensors because all computation is tensor operations.

Why this matters: every data element in deep learning is a tensor. The model input is a tensor, the weights are tensors, the activations are tensors, the gradients are tensors. Understanding tensor shapes is the first step to debugging neural networks. A shape mismatch error is always a tensor with the wrong dimensions being fed to an operation expecting different dimensions.

Why this matters in AI
Scalars, vectors, matrices, and tensors are the data types of machine learning. Every ML algorithm operates on these structures. Understanding the difference between a vector (1D) and matrix (2D) prevents shape errors. Understanding tensor rank enables you to reason about batch dimensions, sequence dimensions, and spatial dimensions.
Common mistakes
  • Confusing a shape (10,) with (10, 1) — a 1D vector and a column vector are different shapes. Many operations produce unexpected results due to this confusion. Use .reshape(-1, 1) to add a dimension.
  • Forgetting the batch dimension — neural networks process batches, not single samples. A model expecting (batch, features) will fail on a single sample (features,). Add a batch dimension: x.unsqueeze(0).
  • Treating tensors as immutable — both NumPy and PyTorch tensors are mutable. Operations like += modify the tensor in place; use + to create a new tensor.

Dot product

Detailed explanation

The dot product (inner product) of two vectors a and b is: a·b = Σ(a_i * b_i) — sum of element-wise products. For vectors a = [1, 2, 3] and b = [4, 5, 6]: a·b = 1*4 + 2*5 + 3*6 = 4 + 10 + 18 = 32. The dot product produces a scalar — a single number. It can also be written as a.T @ b (matrix multiplication of a column vector with a row vector).

Geometric interpretation: a·b = |a| * |b| * cos(θ) where |a| is the magnitude of a, |b| is the magnitude of b, and θ is the angle between them. When θ=0° (same direction): cos(0)=1, dot product is maximum. When θ=90° (perpendicular): cos(90)=0, dot product is 0. When θ=180° (opposite directions): cos(180)=-1, dot product is negative. This geometric meaning makes dot products perfect for measuring similarity.

Cosine similarity uses the normalized dot product: cos(θ) = a·b / (|a| * |b|). By dividing by the magnitudes, we get a value between -1 and +1 regardless of vector scale. cos(θ) = 1 means the vectors point in the same direction (identical meaning), cos(θ) = 0 means orthogonal (unrelated), cos(θ) = -1 means opposite meaning. This is used for: sentence similarity (how similar are two documents?), recommendation (user-item similarity), and nearest neighbor search.

In neural networks, the dot product is the core computation of linear layers and attention. A single neuron computes: output = w·x + b — the dot product of the weight vector and input vector, plus a bias. The output of a linear layer is the matrix product X @ W.T, which computes the dot product of each input vector with each weight vector simultaneously. Self-attention computes similarity between query and key vectors: score = Q · K / sqrt(d_k).

Projection: the dot product projects one vector onto another. proj_a(b) = (a·b / |a|²) * a — the component of b in the direction of a. In PCA, we project data onto principal components (eigenvectors) using dot products. In recommendation systems, the dot product of a user embedding with an item embedding measures how much the user 'aligns with' the item.

Worked example

print(dot)  # also a @ b, also sum(a * b)
Cosine similarity: def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
print(cosine_sim(a, b))  # 0.9746 — nearly identical direction
Batch dot products: X = np.random.randn(100, 50)  # 100 user vectors

Self-attention scores: Q, K = np.random.randn(10, 64), np.random.randn(10, 64)  # 10 tokens, d=64
scores = Q @ K.T / np.sqrt(64)  # (10, 10) — attention score matrix
Why this matters in AI
The dot product is the mathematical heart of deep learning. Every linear layer, every attention mechanism, every embedding similarity is computed with dot products. Understanding it geometrically (similarity as alignment) explains why cosine similarity works for semantic search and why attention scores measure relevance.
Common mistakes
  • Confusing np.dot(a, b) with np.multiply(a, b) — dot product sums the element-wise products; multiply returns element-wise products without summing.
  • Not normalizing vectors before using dot product as similarity — raw dot product is scale-dependent. Use cosine similarity for scale-invariant comparison.
  • Forgetting the 1/sqrt(d) scaling in attention — without it, dot products grow with d and push softmax into saturation (very peaky distributions). The scaling keeps gradients stable.

Matrix multiplication

Detailed explanation

Matrix multiplication is the core computation of neural networks. Every linear layer, attention score, and embedding lookup is matrix multiplication. The result of multiplying matrix A (m×k) by matrix B (k×n) is matrix C (m×n) where C[i,j] = sum over k of A[i,k] × B[k,j]. The dimensions must be compatible: the number of columns in A must equal the number of rows in B. The result has m rows and n columns.

Geometric intuition: matrix multiplication applies a linear transformation. Multiplying a vector v by matrix A produces a new vector Av in a transformed space. A 2×2 matrix can rotate, scale, shear, or reflect 2D vectors. A 3×2 matrix maps 2D vectors to 3D space (projection). In neural networks, each linear layer weight matrix W transforms the input representation into a new, hopefully more task-useful representation.

Computational complexity: matrix multiplication of A (m×k) × B (k×n) requires m×k×n multiply-add operations. For multiplying two 1000×1000 matrices: 10^9 operations. This is why GPUs are essential for deep learning — they have thousands of cores executing these operations in parallel. The O(n^3) complexity of naive matrix multiplication is why reducing matrix dimensions (bottleneck layers) helps.

Broadcasting in practice: numpy and PyTorch extend matrix multiplication to batches via broadcasting. torch.bmm(A, B) multiplies batches of matrices. In transformer attention: queries Q (batch × seq_len × d_k), keys K (batch × seq_len × d_k). Attention scores = Q @ K.transpose(-2,-1) / sqrt(d_k) produces (batch × seq_len × seq_len) — one attention score matrix per sequence position per batch.

Numerical stability: matrix multiplication with very large or very small values can cause overflow/underflow in float32. float16 is particularly prone — transformer attention scores are divided by sqrt(d_k) to prevent softmax saturation from large dot products. Mixed precision training (float16 for most operations, float32 for numerically sensitive ones) provides a good balance of speed and stability.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
import torch
# Manual matrix multiplication
A = np.array([[1,2,3],[4,5,6]])      # 2×3
B = np.array([[7,8],[9,10],[11,12]]) # 3×2
C = A @ B  # 2×2 result
# Verify: C[i,j] = sum(A[i,:] * B[:,j])
print('C[0,0]:', sum(A[0,k]*B[k,0] for k in range(3)))  # = 58
print('numpy @:', C)
# Batched matrix multiplication in PyTorch
batch, seq, d_model, d_k = 2, 5, 8, 4
Q = torch.randn(batch, seq, d_k)
K = torch.randn(batch, seq, d_k)
V = torch.randn(batch, seq, d_model)
# Scaled dot-product attention
scores = Q @ K.transpose(-2,-1) / (d_k ** 0.5)  # (batch, seq, seq)
weights = torch.softmax(scores, dim=-1)
output = weights @ V  # (batch, seq, d_model)
print('Attention output shape:', output.shape)
# Efficiency: einsum vs matmul
import time
A, B = torch.randn(1000,1000), torch.randn(1000,1000)
t0 = time.time(); _ = A @ B; print(f'matmul: {time.time()-t0:.4f}s')
t0 = time.time(); _ = torch.einsum('ij,jk->ik', A, B); print(f'einsum: {time.time()-t0:.4f}s')
Why this matters in AI
Every forward pass and backward pass in a neural network is primarily matrix multiplication. The entire architecture of transformers — Q@K^T for attention, W@x for feed-forward layers, embedding lookups — is matrix operations. Profiling neural network performance always traces back to matrix multiplication efficiency.
Common mistakes
  • Confusing element-wise multiplication (*) with matrix multiplication (@) — A * B multiplies corresponding elements (must be same shape), A @ B is matrix multiplication (inner dimensions must match). In PyTorch, * is element-wise, @ and torch.matmul() are matrix multiplication.
  • Wrong dimension order — remember: (m×k) @ (k×n) = (m×n). The inner dimensions k must match. A (3×5) @ B (5×7) = C (3×7). If you get 'mat1 and mat2 shapes cannot be multiplied', check and transpose the right matrix.
  • Not using GPU for large matrix operations — matrix multiplication on CPU for 4096×4096 matrices takes seconds. On GPU (CUDA), the same operation takes milliseconds. Always use .to('cuda') for large matrix operations in deep learning.

Transpose and inverse intuition

Detailed explanation

Matrix transposition and inversion are fundamental linear algebra operations that appear constantly in ML: in the attention mechanism (QK^T), in linear regression solutions (X^T X)^{-1} X^T y, and in gradient calculations. Understanding them intuitively makes reading ML papers and code much faster.

Transpose (A^T): 'flip a matrix over its diagonal.' The element at row i, column j moves to row j, column i. A (3×5) matrix becomes (5×3) after transposition. In ML: A^T is used to match dimensions for matrix multiplication. The dot product of two vectors u and v is u^T v (or u·v). The outer product of u and v is u v^T (a rank-1 matrix).

Why QK^T in attention: Q is (batch, seq, d_k), K is (batch, seq, d_k). To compute similarity between every query and every key, we need (batch, seq, seq). Q × K^T gives exactly this: for each query position, the dot product with each key position. This is why transposing K is necessary — to align the dimensions for matrix multiplication.

Inverse (A^{-1}): A^{-1} 'undoes' A. A × A^{-1} = I (identity matrix). The ordinary least squares solution to Ax = b is x = A^{-1}b. In practice, you never compute the inverse explicitly (numerical instability and O(n^3) cost). Instead: `np.linalg.solve(A, b)` computes the same result more stably via LU decomposition.

Pseudoinverse (A^+): when A is not square or not invertible (which happens frequently in ML — more features than samples, or correlated features), use the Moore-Penrose pseudoinverse. X^+ = (X^T X)^{-1} X^T is the left pseudoinverse used in linear regression. In numpy: `np.linalg.pinv(X)`. sklearn's LinearRegression uses this under the hood.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
# Transpose
A = np.array([[1, 2, 3], [4, 5, 6]])  # shape (2, 3)
print('A:', A.shape)
print('A^T:', A.T.shape)  # (3, 2)
# Dot product as transpose multiplication
u, v = np.array([1., 2., 3.]), np.array([4., 5., 6.])
print('u . v =', u @ v)       # scalar: 1*4 + 2*5 + 3*6 = 32
print('u^T @ v =', u @ v)    # same result: matrix form of dot product
# Outer product
print('u @ v^T shape:', np.outer(u, v).shape)  # (3, 3) rank-1 matrix
# QK^T in attention
Q = np.random.rand(4, 8, 64)  # batch=4, seq=8, d_k=64
K = np.random.rand(4, 8, 64)
scores = Q @ K.transpose(0, 2, 1)  # (4, 8, 64) @ (4, 64, 8) = (4, 8, 8)
print('Attention scores shape:', scores.shape)  # (4, 8, 8): 8 queries x 8 keys
# Linear regression: normal equations
np.random.seed(42)
n, p = 100, 5
X = np.column_stack([np.ones(n), np.random.randn(n, p)])
beta_true = np.array([2., 1., -0.5, 0.3, 0., -1.])
y = X @ beta_true + np.random.randn(n) * 0.5
# Closed-form solution: beta = (X^T X)^{-1} X^T y
# Method 1: direct inverse (unstable for ill-conditioned matrices)
beta_inv = np.linalg.inv(X.T @ X) @ X.T @ y
# Method 2: solve (numerically stable)
beta_solve = np.linalg.solve(X.T @ X, X.T @ y)
# Method 3: pseudoinverse (most robust)
beta_pinv = np.linalg.pinv(X) @ y
print('True beta:', beta_true)
print('Estimated (solve):', beta_solve.round(3))
Why this matters in AI
Transpose and inverse appear in nearly every ML formula you'll encounter in papers and code. Not understanding them means you must skip over equations rather than reading them fluently. Understanding A^T as 'flip over the diagonal' and the normal equations as 'closed-form linear regression' unlocks the ability to derive, debug, and implement ML algorithms directly from their mathematical formulation.
Common mistakes
  • Computing A^{-1} explicitly and then multiplying — np.linalg.inv(A) @ b is slower and numerically less stable than np.linalg.solve(A, b). Solve uses LU decomposition which avoids amplifying numerical errors. Never compute the explicit inverse unless you need to reuse it for many right-hand sides.
  • Confusing (A^T)^{-1} with (A^{-1})^T — for invertible A: (A^T)^{-1} = (A^{-1})^T. They are equal, but this identity is not obvious. Remembering this prevents errors when rearranging matrix equations.
  • Forgetting that matrix multiplication is not commutative — AB ≠ BA in general. Dimension compatibility: A (m×k) @ B (k×n) = C (m×n). A^T @ B requires A to be (k×m) for A^T to be (m×k). Wrong multiplication order is the most common matrix dimension error.

Eigenvalues intuition

Detailed explanation

Eigenvalues and eigenvectors describe the directions in which a matrix transformation stretches (or compresses) space without rotating it. If A × v = λ × v, then v is an eigenvector of A and λ is its eigenvalue. The matrix A, when multiplied by v, produces a vector in the SAME direction as v but scaled by λ. Positive λ > 1 = stretch, 0 < λ < 1 = compress, λ < 0 = flip direction.

Intuition with examples: consider a 2×2 matrix that represents 'scale x-direction by 3, scale y-direction by 0.5'. Its eigenvectors are (1,0) and (0,1) with eigenvalues 3 and 0.5. For any other direction, the transformation ROTATES the vector as well as scaling it — but these two special directions (eigenvectors) only scale. The eigenvectors are the 'invariant directions' of the transformation.

PCA uses eigendecomposition: the covariance matrix Σ of the data has eigenvectors that point in the directions of maximum variance (principal components) and eigenvalues that equal the variance explained by each direction. The eigenvector with the largest eigenvalue points in the direction of most variance in the data. PCA = sort eigenvectors by eigenvalue and project the data onto the top k.

Eigenvalues in deep learning: the Hessian matrix (second-order derivatives) has eigenvalues that describe the curvature of the loss landscape. Large eigenvalues = sharp curvature = difficult to optimize (gradient descent steps may overshoot). Small eigenvalues = flat landscape. The condition number (max_eigenvalue / min_eigenvalue) measures how 'elongated' the loss landscape is — high condition number = gradient descent converges slowly, motivating adaptive optimizers like Adam.

Spectral methods: Graph neural networks use the eigendecomposition of the graph Laplacian (spectral graph convolution). Word2Vec's PMI matrix eigendecomposition produces word embeddings. PageRank is the dominant eigenvector of a web graph's transition matrix. Understanding eigenvalues opens the door to a deep class of ML algorithms grounded in linear algebra.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
import matplotlib.pyplot as plt
# Eigendecomposition of a covariance matrix
np.random.seed(42)
data = np.random.multivariate_normal([0,0], [[3,2],[2,2]], size=500)
cov = np.cov(data.T)
eigenvalues, eigenvectors = np.linalg.eig(cov)
print('Eigenvalues:', eigenvalues)  # variance along each PC
print('Eigenvectors:\n', eigenvectors)  # principal component directions
# Verify: A @ v = λ @ v
for i in range(2):
    lhs = cov @ eigenvectors[:,i]
    rhs = eigenvalues[i] * eigenvectors[:,i]
    print(f'Eigenvector {i} check: {np.allclose(lhs, rhs)}')
# PCA via eigendecomposition vs sklearn
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
pca.fit(data)
print('sklearn PCA explained variance:', pca.explained_variance_)
print('Matches eigenvalues:', np.allclose(sorted(pca.explained_variance_, reverse=True), sorted(eigenvalues, reverse=True)))
# Variance explained ratio
for i, (ev, ratio) in enumerate(zip(pca.explained_variance_, pca.explained_variance_ratio_)):
    print(f'PC{i+1}: eigenvalue={ev:.3f}, explains {ratio*100:.1f}% of variance')
Why this matters in AI
Eigenvalues underpin PCA, SVD, spectral clustering, and the theory of neural network optimization. When you run PCA to reduce 1000 features to 50, you are computing eigendecomposition of the covariance matrix. When Adam optimizer works better than SGD, the explanation is in the eigenvalue structure of the Hessian.
Common mistakes
  • Confusing eigenvectors with principal components — in PCA, the principal components ARE the eigenvectors of the covariance matrix. But numpy.linalg.eig returns eigenvectors unsorted — sort by eigenvalue magnitude for PCA.
  • Using numpy.linalg.eig for symmetric matrices — symmetric positive-definite matrices (covariance matrices) should use numpy.linalg.eigh (hermitian/symmetric version) which is faster and always returns real eigenvalues. numpy.linalg.eig may return complex numbers for numerical reasons.
  • Interpreting eigenvalue magnitude as 'importance' without normalizing — eigenvalue magnitude depends on the data's scale. If features are not standardized before PCA, features with large variance dominate. Always StandardScale before PCA.

Norm and distance

Detailed explanation

A norm is a function that assigns a non-negative length (magnitude) to a vector. The L1 norm (Manhattan norm) is the sum of absolute values: ||x||_1 = Σ|x_i|. The L2 norm (Euclidean norm) is the square root of the sum of squares: ||x||_2 = sqrt(Σx_i²). The L∞ norm (Chebyshev norm) is the maximum absolute value: ||x||_∞ = max|x_i|. Each norm defines a different geometry and is appropriate for different problems.

Euclidean distance between two points is ||x - y||_2 = sqrt(Σ(x_i - y_i)²). Manhattan distance is ||x - y||_1 = Σ|x_i - y_i|. These are the distance metrics underlying KNN and clustering algorithms. Euclidean distance measures 'as the crow flies' distance; Manhattan measures 'city block' distance. For high-dimensional spaces, Manhattan distance is often more reliable than Euclidean (curse of dimensionality).

L1 and L2 regularization add norm penalties to the loss function. L1 regularization (Lasso): loss + λ*||w||_1 — pushes weights to exactly zero, producing sparse models. L2 regularization (Ridge): loss + λ*||w||_2² — shrinks weights toward zero but rarely to exactly zero, producing small but non-zero weights. L1 is used for feature selection (sparse solutions); L2 prevents any single weight from dominating. ElasticNet combines both.

Normalization in ML: normalizing vectors to unit length (||x||_2 = 1) makes distances between normalized vectors equivalent to cosine similarity, which is angle-based rather than magnitude-based. sklearn's Normalizer applies L2 normalization: each row of X is divided by its L2 norm. This is appropriate before cosine similarity-based operations (text TF-IDF vectors, embedding similarity).

Mahalanobis distance accounts for correlations between features: d_M(x,y) = sqrt((x-y)^T Σ^{-1} (x-y)) where Σ is the covariance matrix. If features are correlated and on different scales, Euclidean distance gives misleading results. Mahalanobis distance is scale-invariant and accounts for correlations — it's equivalent to Euclidean distance after standardization and decorrelation (whitening).

Distance: a = np.array([1,2,3]); b = np.array([4,5,6])

Normalize: from sklearn.preprocessing import normalize

Why this matters in AI
Norms and distances are the mathematical foundation of distance-based ML: KNN, clustering, SVM, embedding similarity, regularization. Understanding why L1 produces sparse solutions and L2 does not explains how regularization controls model complexity. Understanding when to use cosine vs Euclidean distance prevents wrong similarity computations.
Common mistakes
  • Using Euclidean distance on unnormalized high-dimensional vectors — in high dimensions, all points are approximately equidistant (curse of dimensionality). Use cosine similarity instead.
  • Applying L1 when you want smooth solutions — L1 regularization produces discontinuous solutions at 0. For smooth, small-but-non-zero weights, use L2.
  • Computing distances without scaling features first — features on different scales distort distances. Standardize features before computing Euclidean distances.

Cosine similarity

Detailed explanation

Cosine similarity measures the cosine of the angle between two vectors: cos(θ) = (A·B)/(||A|| * ||B||). It ranges from -1 (opposite direction) to +1 (same direction). A cosine similarity of 1 means the vectors point in the same direction (regardless of magnitude). This makes it ideal for text and embedding similarity — two documents about 'machine learning' should be similar whether they are short or long.

Why cosine similarity is preferred for text: a long document about machine learning and a short document about machine learning have different TF-IDF vector magnitudes (the long one has larger values because terms appear more times). But their direction is similar. Cosine similarity normalizes by magnitude — it measures direction (topic) rather than magnitude (length). Euclidean distance would incorrectly find the long document dissimilar from the short one.

In embedding spaces (word2vec, BERT, sentence transformers), semantically similar concepts have high cosine similarity. king - man + woman ≈ queen: the difference vectors are parallel in the embedding space. Semantic search retrieves documents with high cosine similarity to the query embedding. This is how all modern embedding-based retrieval works.

Computing cosine similarity efficiently: for large sets of vectors, precompute normalized embeddings (divide each by its L2 norm), then use matrix multiplication to compute all pairwise dot products: similarities = X_norm @ X_norm.T. Since vectors are normalized, dot products equal cosine similarities. This vectorized approach is 100–1000× faster than computing similarities in a Python loop.

Cosine similarity vs dot product: normalized dot product = cosine similarity. For semantic search, normalizing embeddings before computing dot products enables using efficient MIPS (Maximum Inner Product Search) algorithms — the same algorithms used in FAISS and vector databases — for cosine similarity search.

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

a = np.array([1.0, 0.0, 1.0])  # document about topic A
b = np.array([2.0, 0.0, 2.0])  # same topic, longer document
c = np.array([0.0, 1.0, 0.0])  # different topic
print(cosine_similarity(a, b))  # 1.0 — same direction
print(cosine_similarity(a, c))  # 0.0 — orthogonal

Why this matters in AI
Cosine similarity is the universal metric for semantic similarity in NLP and embedding-based search. Understanding it enables you to build semantic search, find similar users/items in recommendation systems, and cluster documents by topic.
Common mistakes
  • Using Euclidean distance instead of cosine similarity for text embeddings — Euclidean distance penalizes different magnitudes (document lengths) rather than topic similarity.
  • Not normalizing before using dot products as similarities — raw dot products are scale-dependent. Normalize to unit vectors first.
  • Expecting cosine similarity to work for vectors with many near-zero dimensions — sparse vectors (TF-IDF) need dot product computation on only non-zero dimensions for efficiency. Use sparse matrix operations.
