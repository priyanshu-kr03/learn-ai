Neural networks

Detailed explanation

A neural network is a function that maps input data to output predictions through a series of layers. Each layer applies a linear transformation (multiply by weights, add bias) followed by a nonlinear activation function. Stacking many layers allows the network to learn hierarchical representations — early layers detect simple patterns, later layers detect complex abstractions.

The biological neuron analogy is loose but useful: inputs arrive through connections (synapses), are weighted by synapse strength, summed in the cell body, and fire an output signal if the sum exceeds a threshold. In an artificial neuron, inputs are numbers, weights are learnable parameters, the sum is a dot product, and the threshold is replaced by a smooth activation function.

A fully connected (dense) neural network with one hidden layer can approximate any continuous function if the layer is wide enough — this is the Universal Approximation Theorem. In practice, depth (more layers) is more efficient than width (wider layers) for learning complex structured representations like images, text, and audio.

Neural networks learn by minimizing a loss function using gradient descent and backpropagation. The loss measures prediction error. Backpropagation computes how much each weight contributed to the error using the chain rule. The optimizer then nudges each weight slightly in the direction that reduces error.

Architecture: input layer (4 features) → hidden layer (8 neurons, ReLU) → output layer (1 neuron, sigmoid).

Why this matters in AI
Neural networks are the foundation of deep learning. Understanding how layers, weights, activations, and backpropagation work together lets you debug training failures and design better architectures.
Common mistakes
  • Using linear activation in hidden layers — without nonlinearity, any depth collapses to a single linear transformation.
  • Not initializing weights properly — all-zero initialization causes all neurons to learn the same thing (symmetry breaking failure).
  • Using a classification loss (cross-entropy) for regression or vice versa — loss function must match the output type.

Perceptron

Detailed explanation

The perceptron is the simplest neural unit, invented by Frank Rosenblatt in 1957. It takes a vector of inputs, multiplies each by a learnable weight, sums the results, adds a bias, and passes the total through a step function to produce a binary output (0 or 1). It is the building block from which modern neural networks are constructed.

A single perceptron can learn any linearly separable binary classification problem. It can separate AND and OR, but not XOR, because XOR is not linearly separable — no single line divides the two classes. This limitation, highlighted in Minsky and Papert's 1969 book, contributed to the first 'AI winter'.

The perceptron learning rule updates weights when a prediction is wrong: W = W + learning_rate * (actual - predicted) * input. This rule guarantees convergence for linearly separable data. For non-linearly separable data, it loops forever without converging — which is why we add hidden layers and use gradient descent instead.

Modern neural networks replace the step function with smooth activations (sigmoid, ReLU) that allow gradient computation. This single change from step function to smooth activation makes backpropagation possible and unlocks learning for non-linear problems.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

Input: x = [0.5, 0.8], weights: w = [0.3, -0.2], bias: b = 0.1.
Weighted sum: z = 0.5*0.3 + 0.8*(-0.2) + 0.1 = 0.15 - 0.16 + 0.1 = 0.09.
Step function: if z >= 0 → predict 1. Since 0.09 >= 0 → output = 1.
If true label = 0: error = 0 - 1 = -1. Update: w = w + 0.1 * (-1) * x.
New weights: [0.3 - 0.05, -0.2 + 0.08] = [0.25, -0.12].
Why this matters in AI
The perceptron is the atom of neural networks. Understanding it — what it computes, what it cannot compute, and why — builds the intuition for why we need hidden layers and nonlinear activations.
Common mistakes
  • Expecting a single perceptron to solve non-linear problems — it cannot separate XOR regardless of training time.
  • Confusing the perceptron update rule with gradient descent — the perceptron rule is discrete and only fires on misclassification.
  • Skipping the perceptron and jumping to deep networks — its simplicity makes weights, bias, and the learning process transparent.

Layers and weights

Detailed explanation

A neural network layer is a collection of neurons that all receive the same input and each produce one output. The input layer receives the raw features. Hidden layers learn intermediate representations. The output layer produces the final prediction. Each layer is defined by a weight matrix W and a bias vector b.

The weight matrix W for a layer that takes n inputs and produces m outputs has shape (m, n). Each row of W is the weights for one neuron. The bias vector b has m values, one per neuron. The layer computes: output = activation(W @ input + b), where @ is matrix multiplication.

Weights are initialized randomly (e.g. He initialization for ReLU, Xavier for tanh) to break symmetry and allow different neurons to learn different patterns. During training, the optimizer updates every weight using its gradient. After training, the weights encode everything the model has learned about the patterns in the data.

The number of layers and neurons per layer defines model capacity. A shallow wide network and a deep narrow network with the same total parameters often learn differently. Depth generally wins for structured data (images, language) because it allows hierarchical feature learning.

Why this matters in AI
Weight matrices are what neural networks learn. Understanding their shape and initialization prevents shape errors and helps diagnose training problems.
Common mistakes
  • Initializing all weights to zero — every neuron computes the same thing and learns identically (symmetry failure).
  • Using the same initialization for all activation types — He init is for ReLU, Xavier for sigmoid/tanh.
  • Not tracking weight magnitudes — very large or very small weights indicate training instability.

Activation functions

Detailed explanation

Activation functions introduce nonlinearity into neural networks. Without them, any stack of linear layers is mathematically equivalent to a single linear layer — no depth can be gained. The activation function applied after each linear transformation allows the network to learn nonlinear decision boundaries.

The choice of activation function affects training speed, gradient flow, and output range. Sigmoid squashes any value to (0, 1) — useful for binary output probabilities but causes vanishing gradients in deep networks. ReLU (Rectified Linear Unit) is zero for negative inputs and linear for positive inputs — fast, simple, and works well in practice for hidden layers. GELU is a smooth version of ReLU used in BERT and GPT. Softmax converts a vector of scores into probabilities that sum to 1 — used in the output layer for multiclass classification.

The dying ReLU problem occurs when a neuron always receives negative input and its gradient is always zero — it stops learning entirely. Leaky ReLU and ELU address this by allowing a small negative gradient. GELU is smooth everywhere and has become the default in transformer models.

Output layer activation must match the task: sigmoid for binary classification, softmax for multiclass, linear (no activation) for regression. Using the wrong output activation is a common bug that produces nonsensical predictions.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

Sigmoid: f(2.0) = 1/(1+e^{-2}) ≈ 0.88. f(-2.0) ≈ 0.12. Gradient near 0 for large inputs.
ReLU: f(2.0) = 2.0. f(-2.0) = 0.0. Gradient = 1 for positive, 0 for negative.
Softmax on [2.0, 1.0, 0.5]: exp values = [7.39, 2.72, 1.65], sum=11.76 → [0.63, 0.23, 0.14].
Test: does your model use linear activation in a hidden layer? Replace it with ReLU and watch training improve.
Why this matters in AI
Wrong activation functions are a silent killer — a linear hidden layer or a sigmoid output for regression produces wrong results with no obvious error message.
Common mistakes
  • Using sigmoid in hidden layers of deep networks — gradient vanishes and early layers stop learning.
  • Using softmax in a hidden layer — it introduces coupling between neuron outputs that disrupts learning.
  • Forgetting activation on the output layer for binary classification — raw logits are not probabilities.

Sigmoid tanh ReLU GELU softmax

Detailed explanation

Sigmoid maps any real number to the range (0, 1): f(x) = 1/(1+e^{-x}). Output near 1 means high confidence in the positive class. Its gradient peaks at 0.25 when x=0 and approaches 0 for large |x|. This gradient saturation makes sigmoid unsuitable for deep hidden layers — backpropagated gradients shrink to near-zero by the time they reach early layers.

Tanh (hyperbolic tangent) maps to (-1, 1): f(x) = (e^x - e^{-x})/(e^x + e^{-x}). Zero-centered unlike sigmoid, which helps gradient flow slightly. Still saturates at extremes. Used in LSTM gates and some older architectures. Gradient peaks at 1.0 when x=0 and decays at extremes.

ReLU (Rectified Linear Unit) is f(x) = max(0, x). Linear for positive inputs, zero for negative. Its gradient is either 1 (positive input) or 0 (negative input) — no saturation on the positive side. Computationally cheap. Default choice for deep hidden layers. Variants: Leaky ReLU allows small negative gradient; ELU smooths the negative region; SiLU/Swish is x * sigmoid(x) and used in modern vision models.

GELU (Gaussian Error Linear Unit) is approximately x * Φ(x) where Φ is the Gaussian CDF. Smooth everywhere, including at zero. Used in BERT, GPT, and most modern transformer architectures. Empirically outperforms ReLU on NLP tasks. Softmax for a vector z: f(z_i) = exp(z_i) / sum(exp(z_j)). Produces a probability distribution. Used only in output layers for classification, and in attention score normalization.

Worked example

Sigmoid: f(0) = 0.5, f(2) ≈ 0.88, f(-2) ≈ 0.12, f(10) ≈ 1.0 (saturated).
Tanh: f(0) = 0, f(2) ≈ 0.96, f(-2) ≈ -0.96, f(10) ≈ 1.0 (saturated).
ReLU: f(0) = 0, f(2) = 2, f(-2) = 0, f(10) = 10 (no saturation for positive).
GELU: f(0) = 0, f(2) ≈ 1.95, f(-2) ≈ -0.05 (smooth near zero).
Rule of thumb: ReLU or GELU for hidden layers, sigmoid for binary output, softmax for multiclass output, linear for regression.
Why this matters in AI
Choosing the wrong activation is one of the most common causes of training failure. Understanding each function's range, gradient, and saturation behavior makes architecture design deliberate rather than guesswork.
Common mistakes
  • Using sigmoid in all hidden layers of a network deeper than 3 layers — gradient vanishing halts learning in early layers.
  • Forgetting that ReLU can produce dead neurons — if learning rate is too high, some neurons always output 0 and never recover.
  • Using softmax followed by cross-entropy via PyTorch's CrossEntropyLoss — that function expects raw logits, not softmax outputs, and applying softmax first causes numerical instability.

Forward propagation

Detailed explanation

Forward propagation is the process of passing an input through a neural network layer by layer to produce a prediction. Starting from the input, each layer computes a linear transformation followed by an activation function. The final layer's output is the model's prediction. No weights change during forward propagation — it is a read-only pass.

For a 2-layer network: z1 = W1 @ x + b1, h1 = ReLU(z1), z2 = W2 @ h1 + b2, ŷ = sigmoid(z2). Each intermediate value must be stored during forward pass so backpropagation can use it when computing gradients. This is why training uses more memory than inference.

Batch processing makes forward propagation efficient. Instead of passing one example at a time, pack N examples into a matrix of shape (N, features). The matrix multiplications then naturally produce outputs for all N examples in one GPU operation. Batch size affects both training speed and memory usage.

During inference (making predictions on new data), forward propagation is all that runs. Gradient computation, optimizer steps, and dropout are disabled. PyTorch uses torch.no_grad() to skip building the computation graph during inference, saving memory and time.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

Network: input (2) → hidden (3, ReLU) → output (1, sigmoid).
Input: x = [[1.0, 0.5]] — shape (1, 2).
z1 = x @ W1.T + b1 — shape (1, 3). Apply ReLU → h1.
z2 = h1 @ W2.T + b2 — shape (1, 1). Apply sigmoid → ŷ.
ŷ is a probability. Compare with label y to compute loss.
Why this matters in AI
Forward propagation is the inference engine of all neural networks. Getting shapes and activations right here prevents errors in the backward pass.
Common mistakes
  • Forgetting to store intermediate activations during training — backpropagation requires them to compute gradients.
  • Running forward pass under torch.no_grad() during training — this disables gradient tracking and backpropagation will fail.
  • Mixing up batch size and feature dimension in matrix shapes — shape errors here are the most common PyTorch bug.

Loss functions

Detailed explanation

A loss function measures how wrong a model is on an example or batch. It turns prediction quality into a number that the optimizer can minimize. Without a loss function, the model has no clear learning signal.

Different tasks need different losses. Regression often uses mean squared error or mean absolute error because the target is a number. Classification often uses cross-entropy because the model predicts probabilities over classes.

For regression, if the true house price is 300,000 and the model predicts 270,000, the error is 30,000. Squared error makes that 900,000,000, which strongly punishes large mistakes. Mean absolute error would count it as 30,000, which is easier to interpret but gives a different optimization behavior.

Choosing a loss function is not just a math choice. It tells the model what kind of mistake matters. If large errors are very costly, squared error may be appropriate. If you want robustness to outliers, absolute error may be better.

Why this matters in AI
Math intuition helps you debug models. When a metric, gradient, probability, or similarity score behaves strangely, you can reason about the cause instead of guessing.
Common mistakes
  • Memorizing formulas without knowing what each term represents.
  • Skipping units, shapes, or assumptions.
  • Using a metric because it is popular instead of because it matches the problem.

Backpropagation

Detailed explanation

Backpropagation computes the gradient of the loss with respect to every weight in the network by applying the chain rule of calculus from the output layer back to the input layer. It answers: for each weight, if we increase it by a tiny amount, how much does the loss change? That answer is the gradient, and the optimizer uses it to update the weight.

The backward pass starts at the loss. It computes dLoss/d(output), then dLoss/d(pre-activation), then dLoss/dW and dLoss/db for the last layer, then passes the gradient backward through the activation function to the previous layer — and so on until the first layer. Each layer only needs to know its local gradient and the gradient flowing in from the layer above.

Deep frameworks (PyTorch, TensorFlow) automate this via automatic differentiation. When you call loss.backward() in PyTorch, it traverses the computation graph built during the forward pass and computes all gradients automatically. The gradients are stored in the .grad attribute of each parameter tensor.

Numerical gradient checking is a debugging technique: compute the gradient analytically and also estimate it numerically by perturbing each weight slightly and measuring the change in loss. If they match within a small tolerance, the backward pass is correct. This is how you verify a custom layer's gradient implementation.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

Forward: x → z1 = W1@x + b1 → h1 = ReLU(z1) → z2 = W2@h1 + b2 → ŷ = sigmoid(z2).
Loss: L = binary_cross_entropy(ŷ, y).
dL/dŷ = (ŷ - y) / (ŷ*(1-ŷ)) [from BCE derivative].
dL/dz2 = dL/dŷ * sigmoid'(z2) = ŷ - y [for BCE + sigmoid, gradient simplifies cleanly].
dL/dW2 = dL/dz2 @ h1.T; dL/db2 = dL/dz2. Continue backward through ReLU and first layer.
Why this matters in AI
Backpropagation is how networks learn. Understanding it lets you debug vanishing gradients, verify custom layers, and understand why certain architectures fail.
Common mistakes
  • Calling optimizer.zero_grad() after loss.backward() instead of before — gradients accumulate across batches and produce wrong updates.
  • Not calling loss.backward() before optimizer.step() — weights update using zero or stale gradients.
  • Expecting backprop to work through non-differentiable operations like argmax — use straight-through estimators or rethink the architecture.

Optimizers

Detailed explanation

An optimizer updates model weights using computed gradients to minimize the loss. The simplest optimizer is Stochastic Gradient Descent (SGD): weight = weight - learning_rate * gradient. It updates parameters using the gradient from a mini-batch, not the full dataset, which introduces noise that can escape sharp local minima but also makes training noisy.

Momentum helps SGD move more smoothly by accumulating a velocity vector that dampens oscillations. The update direction incorporates both the current gradient and the accumulated history of past gradients. Nesterov momentum looks one step ahead — it computes the gradient at the expected next position rather than the current one.

Adam (Adaptive Moment Estimation) is the default optimizer for most deep learning tasks. It maintains per-parameter learning rates that adapt based on the first moment (mean of gradients) and second moment (variance of gradients). Parameters with consistently large gradients get smaller effective learning rates; parameters with small gradients get larger ones. AdamW fixes a weight decay implementation bug in Adam and is the standard for training transformers.

Learning rate scheduling adjusts the learning rate during training. Common schedules: cosine annealing (decreases to zero following a cosine curve), warmup (ramp up from near-zero in early steps to prevent instability), plateau reduction (reduce when validation loss stops improving). Most modern training recipes use warmup + cosine decay.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

SGD: lr=0.01, gradient=0.5 → weight_new = weight - 0.01*0.5 = weight - 0.005.
SGD with momentum (β=0.9): v = 0.9*v_prev + 0.1*gradient; weight -= lr * v.
Adam: m = 0.9*m + 0.1*g; v = 0.999*v + 0.001*g^2; m̂ = m/(1-0.9^t); v̂ = v/(1-0.999^t); weight -= lr * m̂/(sqrt(v̂)+ε).
Rule: start with Adam(lr=3e-4) for most problems. Switch to SGD+momentum for fine-tuning vision models where Adam may overfit.
Why this matters in AI
The optimizer and learning rate schedule have as much impact on final accuracy as the model architecture. Using the wrong optimizer or failing to tune the learning rate is a common cause of poor results.
Common mistakes
  • Not zeroing gradients before each backward pass — accumulated gradients from previous batches corrupt the current update.
  • Using Adam with the default weight_decay=0 for transformers — use AdamW with weight_decay=0.01 to prevent overfitting.
  • Setting learning rate too high — loss will spike or NaN will appear. Too low — training converges very slowly or stops.

Dropout

Detailed explanation

Dropout is a regularization technique that randomly sets a fraction of neuron outputs to zero during each training step. With dropout rate p=0.5, each neuron has a 50% chance of being zeroed out for that mini-batch. The neuron is excluded from that forward and backward pass — it contributes no output and receives no gradient update.

Dropout forces the network to learn redundant representations. No single neuron can rely on specific other neurons always being present, so the network develops more robust features. At inference time, dropout is disabled and all neurons are active, but their outputs are scaled down by (1-p) to account for the fact that they were dropped during training — or equivalently, during training the active neurons are scaled up by 1/(1-p) (inverted dropout).

The dropout rate is a hyperparameter. Typical values: 0.1–0.2 for recurrent layers, 0.3–0.5 for fully connected layers. Too high a dropout rate slows learning and underfits. Dropout is not typically used in convolutional layers — spatial dropout (dropping entire channels) is used instead. In transformer models, dropout is applied after attention and feed-forward layers.

Dropout makes training stochastic — different neurons drop each time, so the same input produces a slightly different output on each forward pass. This means training loss is noisier than validation loss (where dropout is off). Use eval mode (model.eval()) during validation and inference to disable dropout.

Worked example

Why this matters in AI
Dropout is one of the most effective and simplest regularizers. A model trained with dropout often generalizes significantly better than the same architecture without it.
Common mistakes
  • Forgetting to call model.eval() before validation or inference — dropout remains active and makes predictions stochastic and artificially low.
  • Applying high dropout (>0.5) to early layers — early layers learn basic features; over-regularizing them prevents the network from learning anything useful.
  • Using dropout as a substitute for other regularization — it works best combined with appropriate model size, weight decay, and data augmentation.

Batch normalization

Detailed explanation

Batch normalization (BatchNorm) normalizes the inputs to a layer across the mini-batch dimension so they have zero mean and unit variance, then applies learnable scale (γ) and shift (β) parameters. This stabilizes training by reducing internal covariate shift — the tendency for layer inputs to shift distribution as earlier layers' weights change.

For each feature dimension, BatchNorm computes the mean and variance across the current mini-batch, normalizes, then scales and shifts. The running mean and variance across all batches are tracked during training and used during inference (since there may not be a batch to compute statistics from at inference time).

BatchNorm enables higher learning rates, reduces sensitivity to initialization, and acts as a mild regularizer. Before BatchNorm (2015), very deep networks were hard to train. After BatchNorm, 100+ layer networks became trainable. It is applied after the linear transformation and before the activation function in most architectures: Linear → BatchNorm → ReLU.

Layer Normalization (LayerNorm) normalizes across the feature dimension for a single example, not across the batch. It works well for NLP/transformers where batch size can be 1 or variable. BERT and GPT use LayerNorm. BatchNorm is preferred for CNNs and vision models where batch size is larger.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

Mini-batch of 4 values for one feature: [2, 4, 6, 8].
Mean = (2+4+6+8)/4 = 5. Variance = mean of squared differences = 5.
Normalized: [(2-5)/√5, (4-5)/√5, (6-5)/√5, (8-5)/√5] = [-1.34, -0.45, 0.45, 1.34].
Scale and shift with learnable γ=1, β=0: output = normalized values (unchanged).
During inference: use running mean and variance accumulated during training, not the current batch.
Why this matters in AI
BatchNorm made training deep networks reliable. Without it, careful initialization and very low learning rates were required. With it, most architectures train stably.
Common mistakes
  • Using BatchNorm with batch size 1 — with one example, mean and variance are trivial and normalization is meaningless. Use LayerNorm instead.
  • Forgetting to track running statistics — at inference, BatchNorm must use training statistics, not compute new ones from a single test example.
  • Applying BatchNorm after the activation — standard placement is before activation (Linear → BN → ReLU).

Vanishing gradients

Detailed explanation

Vanishing gradients occur when gradients shrink exponentially as they propagate backward through many layers. By the time the gradient reaches the first layer, it may be so small (10^{-10} or smaller) that the weights there essentially do not update — the early layers stop learning. This was the main reason deep networks were untrainable before around 2012.

The primary cause is activation functions that saturate: sigmoid and tanh output values near 0 or 1 (or -1) when inputs are large, and their gradients in those regions are nearly zero. Backprop multiplies gradients layer by layer; multiplying many small numbers produces a very small number. With 10 sigmoid layers, the gradient can shrink by a factor of (0.25)^10 ≈ 10^{-6}.

Solutions: ReLU and its variants (Leaky ReLU, GELU) have gradient 1 for positive inputs and do not saturate on the positive side. BatchNorm and LayerNorm prevent extreme pre-activation values that trigger saturation. Residual connections (skip connections in ResNets) allow gradients to flow directly from output to earlier layers bypassing intermediate transformations. Better initialization (He, Xavier) keeps activations in the non-saturating range early in training.

The opposite problem, exploding gradients, occurs in RNNs when gradients grow exponentially. Gradient clipping caps the gradient norm at a maximum value (e.g., 1.0) before the optimizer step. If the gradient norm exceeds the clip threshold, all gradients are scaled down proportionally.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

10-layer sigmoid network. Sigmoid gradient at saturation ≈ 0.01.
Gradient reaching layer 1: 0.01^10 = 10^{-20}. Learning rate 0.001 × 10^{-20} → effectively zero update.
Fix 1: Replace sigmoid with ReLU. Gradient = 1 for positive activations. No shrinkage.
Fix 2: Add residual connections. Gradient path: loss → skip → early layer. Not multiplied by intermediate activations.

Why this matters in AI
Vanishing gradients are why ReLU replaced sigmoid, why ResNets became the standard for image models, and why LSTMs were designed to handle long sequences better than vanilla RNNs.
Common mistakes
  • Building a deep network with all sigmoid hidden layers — this will not train effectively beyond 3–4 layers.
  • Ignoring gradient norms during training — log gradient norms per layer to diagnose vanishing or exploding gradients early.
  • Using very small weight initialization in deep networks — even with ReLU, if weights are too small, pre-activations are all near zero and gradients shrink.
Practice task
Build a 10-layer network with sigmoid activations in NumPy. After one backward pass, print the gradient norm at each layer. Then rebuild with ReLU. Compare gradient norms and show the vanishing/stable difference clearly.
