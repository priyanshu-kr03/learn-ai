NumPy arrays

Detailed explanation

NumPy is the foundational library for numerical computing in Python. Its core data structure is the ndarray (n-dimensional array): a contiguous block of memory holding elements of a single data type. The key advantage over Python lists: NumPy operations run in compiled C code and operate on entire arrays without Python for-loops, making them typically 50–100× faster on large datasets.

An ndarray is described by its shape (tuple of dimension sizes), dtype (data type: float32, float64, int64, bool, etc.), and itemsize (bytes per element). A 3D array of shape (100, 28, 28) represents 100 grayscale 28×28 images. All elements share one dtype — NumPy is homogeneous, unlike Python lists which can mix types.

Creating arrays: np.array([1,2,3]) from a Python list, np.zeros((3,4)) for all-zero array, np.ones((2,2)) for all-one, np.arange(0,10,2) for [0,2,4,6,8], np.linspace(0,1,5) for 5 equally spaced values from 0 to 1, np.random.randn(100,3) for 100 samples of 3 normal random features. Understanding what shape you need before calling a function prevents most shape errors.

Universal functions (ufuncs) operate element-wise on entire arrays without loops: np.sqrt(arr), np.exp(arr), np.log(arr), np.abs(arr). Arithmetic operators (+,-,*,/) also work element-wise: arr1 + arr2 adds corresponding elements, assuming compatible shapes (broadcasting handles mismatched shapes). np.sum(arr, axis=0) sums along the first axis (columns for 2D); axis=1 sums across the second axis (rows for 2D).

Memory layout matters for performance. C-contiguous (row-major, default) arrays are fast when accessing rows sequentially. Operations that create views (slicing, transpose) don't copy data. np.copy(arr) creates an independent copy. arr.reshape(new_shape) creates a view if possible, a copy if not. Always check whether an operation produces a view or a copy when debugging unexpected shared-state mutations.

Why this matters in AI
NumPy is the backbone of all Python numerical computing. pandas, scikit-learn, PyTorch, and TensorFlow all store and process data as NumPy arrays or array-like objects. Fluency with NumPy prevents shape errors and enables you to write vectorized code 50–100× faster than Python loops.
Common mistakes
  • Using Python for-loops over array elements — np.sum(arr) is 100× faster than sum(arr[i] for i in range(len(arr))).
  • Forgetting that slice indexing creates a view, not a copy — modifying arr[0:3] modifies the original array. Use .copy() to be safe.
  • Mixing dtypes accidentally — np.array([1, 2.0, 3]) creates float64, not int64. Be explicit: np.array([1,2,3], dtype=np.float32).

Shape and dtype

Detailed explanation

NumPy arrays have two key attributes: shape and dtype. Shape is a tuple describing the array's dimensions: a 1D array of 100 elements has shape (100,). A 2D matrix with 1000 rows and 50 features has shape (1000, 50). A batch of 32 RGB images of size 224×224 has shape (32, 3, 224, 224) — batch × channels × height × width. Understanding shapes is critical in deep learning where shape mismatches are the #1 source of errors.

dtype (data type) determines how values are stored in memory: int32 (32-bit integer — 4 bytes per element), int64 (64-bit integer — 8 bytes), float32 (32-bit float — 4 bytes — standard for ML), float64 (64-bit float — 8 bytes — default NumPy), bool (1 byte), uint8 (8-bit unsigned integer — used for image pixel values 0-255). Choosing the right dtype affects memory usage and computation speed significantly.

float32 vs float64: GPU computation is optimized for float32 (also called single precision). Training neural networks in float64 is 2× slower on GPU and uses 2× more memory with negligible accuracy benefit. Always use float32 for neural network training: model = model.float() or X_tensor = torch.tensor(X, dtype=torch.float32). sklearn uses float64 by default for scikit-learn models.

Reshaping: array.reshape(new_shape) returns a view (or copy) with a new shape but the same data. arr = np.arange(12).reshape(3, 4) creates a 3×4 matrix. -1 in reshape means 'infer this dimension': arr.reshape(-1, 4) automatically computes the first dimension. arr.reshape(-1) flattens to 1D. This is used constantly in deep learning: batch of images (32, 224, 224, 3) → flatten for FC layer (32, 150528).

Array concatenation and stacking: np.concatenate([a, b], axis=0) joins arrays along an existing axis. np.stack([a, b], axis=0) creates a new axis. np.vstack([a, b]) is equivalent to concatenate along axis=0 for 2D arrays. np.hstack([a, b]) concatenates along axis=1. These operations are used to combine batches, merge features from different sources, and construct training arrays from per-sample computations.

Why this matters in AI
Shape and dtype are the first things to check when debugging NumPy/PyTorch errors. A shape mismatch causes an immediate crash with a cryptic error. Understanding shapes enables you to track data through complex pipelines and diagnose dimension-related bugs instantly.
Common mistakes
  • Confusing shape (1000,) with (1000, 1) — a 1D array and a column vector are different. Many sklearn functions expect (n_samples,) labels but (n_samples, n_features) features. Use .reshape(-1, 1) to add a dimension.
  • Forgetting to convert dtypes before computation — mixing int and float arrays causes unintended integer division. Convert to float explicitly.
  • Not checking shapes before expensive operations — verify shapes match before matrix multiplication. A (100, 50) @ (100, 50) fails; (100, 50) @ (50, 100) succeeds.

Indexing and slicing

Detailed explanation

NumPy indexing and slicing provide powerful ways to access and modify array elements. Basic indexing: arr[2] (element at index 2), arr[2, 3] (element at row 2, column 3), arr[-1] (last element), arr[-1, :] (last row). Slicing: arr[1:5] (elements at indices 1,2,3,4), arr[:, 2] (all rows, column 2), arr[::2] (every other element), arr[::-1] (reversed).

Boolean (fancy) indexing: create a boolean mask array and use it to select elements. mask = X[:, 0] > 0.5; X[mask] returns all rows where the first feature > 0.5. This is one of the most powerful NumPy patterns — you can filter, modify, or replace values in a single expression: X[X < 0] = 0 replaces all negative values with 0 (ReLU operation).

Integer array indexing: use an array of indices to select specific elements. idx = np.array([0, 3, 7]); X[idx] selects rows 0, 3, and 7. This is used for mini-batch sampling: batch_idx = np.random.choice(n, batch_size, replace=False); X_batch = X[batch_idx]. Combined with argsort for top-K selection: top_k_idx = np.argsort(scores)[-k:][::-1]; top_k = X[top_k_idx].

Slices return views, not copies — modifying a slice modifies the original array. arr = np.array([1,2,3,4,5]); view = arr[1:4]; view[0] = 99; print(arr) → [1,99,3,4,5]. This is a performance optimization (no memory copy) but can cause unexpected mutations. Use .copy() to create an independent copy: independent = arr[1:4].copy().

Multi-dimensional slicing patterns: arr[:, :, 0] selects the first channel of all images in a batch. arr[:batch_size] gets the first batch. arr[train_mask, :] selects training rows. These patterns appear constantly in ML code — extracting a feature column, selecting a batch, splitting training from validation data.

Why this matters in AI
Efficient data extraction and manipulation using indexing is fundamental to NumPy-based ML. Feature selection (arr[:, feature_cols]), batch extraction (arr[batch_idx]), and data filtering (arr[mask]) are operations you perform hundreds of times in any ML project. Mastering them enables writing vectorized, efficient code.
Common mistakes
  • Expecting slices to copy data — arr[::2] is a view; modifying it modifies the original. Use .copy() when you need an independent copy.
  • Using Python loops instead of boolean indexing — for i in range(len(arr)): if arr[i] > 0: ... is O(n) Python loops. arr[arr > 0] is O(n) in optimized C. The second is 10–100× faster.
  • Off-by-one indexing — Python is 0-indexed; arr[0:3] returns elements at indices 0, 1, 2 (not 3). The end index in slices is exclusive.

Filtering arrays

Detailed explanation

NumPy boolean indexing (fancy indexing) is one of the most powerful and commonly used features for filtering data. A boolean array (mask) of the same length as the array is used to select elements: arr[mask] returns all elements where mask is True. This vectorized operation is orders of magnitude faster than a Python loop.

Creating boolean masks: arr > 5 produces a boolean array where each element is True if arr[i] > 5. Multiple conditions: (arr > 0) & (arr < 10) — use & (and), | (or), ~ (not), not Python's and/or/not which operate on scalars. Always wrap each condition in parentheses due to operator precedence: (a > 0) & (b < 5), NOT a > 0 & b < 5 (& has higher precedence than >).

np.where(): vectorized if-else. np.where(condition, value_if_true, value_if_false). Example: np.where(ages >= 18, 'adult', 'minor') creates an array of strings based on the age condition. np.where(condition) without value arguments returns the indices where condition is True — useful for finding positions.

pandas filtering: df[df['age'] > 30] returns rows where age > 30. Multiple conditions: df[(df['age'] > 30) & (df['income'] > 50000)]. .isin(): df[df['city'].isin(['NYC', 'LA', 'Chicago'])]. .query(): df.query('age > 30 and income > 50000') — often cleaner for complex conditions. .loc[mask, columns]: filter rows and select columns simultaneously.

Boolean indexing for ML: selecting training samples by date range (df[df['date'] < split_date]), filtering out outliers (df[df['income'] < df['income'].quantile(0.99)]), selecting minority class samples (df[df['label'] == 1]), extracting misclassified samples (X_test[y_pred != y_test]). Boolean indexing enables clean, readable data manipulation without loops.

Worked example

Why this matters in AI
Boolean filtering is the workhorse of data manipulation in ML. Selecting training data by time range, filtering outliers, extracting minority class samples, finding misclassified examples — all use boolean indexing. Mastering it replaces dozens of slow, verbose Python loops.
Common mistakes
  • Using Python's and/or instead of &/| for numpy/pandas — 'a > 0 and b < 5' raises ValueError for arrays. Use '(a > 0) & (b < 5)'. The same for pandas Series conditions.
  • Forgetting parentheses around each condition — arr > 0 & arr < 5 evaluates as arr > (0 & arr) < 5 due to precedence. Always use: (arr > 0) & (arr < 5).
  • Modifying arrays in-place via boolean indexing when a copy is needed — arr[mask] returns a copy; modifying it doesn't change arr. Use arr[mask] = new_values to modify in-place.

Broadcasting and vectorization

Detailed explanation

Vectorization means expressing operations on whole arrays without explicit Python for-loops. Instead of for i in range(n): result[i] = a[i] + b[i], you write result = a + b. The addition is executed in compiled C code across all elements simultaneously. For arrays of 1 million elements, the vectorized version is typically 50–200× faster than the Python loop.

Broadcasting is NumPy's set of rules for performing operations between arrays of different but compatible shapes, without copying data. When you add a shape-(3,) array to a shape-(4,3) array, NumPy broadcasts the smaller array to match the larger one: conceptually, the (3,) array is repeated 4 times along a new first dimension to create (4,3), then added element-wise. No actual copy is made — broadcasting is a view trick.

Broadcasting rules: two arrays are compatible for an operation if, for each dimension (aligned from the right), the sizes are equal OR one of them is 1. NumPy automatically expands size-1 dimensions. Shape (100,1) + shape (1,50) → shape (100,50): every pair of rows and columns is computed without copying data.

Vectorizing ML computations: instead of looping over training examples, stack them in a matrix. Linear regression prediction: y_pred = X @ w + b computes predictions for all N examples simultaneously. Gradient descent update: w -= lr * (X.T @ (y_pred - y)) / N — this is the full gradient computed in three lines of vectorized code. Compare: writing this as Python loops would take 10 lines and run 100× slower.

When vectorization is not possible: some algorithms inherently have sequential dependencies (recurrent computations, tree traversal, dynamic programming). In these cases, write the loop in Cython, Numba (jit-compiled Python), or call into a library that handles it in C. For large-scale scientific computing, look at JAX's jit + vmap for GPU-accelerated vectorization.

Why this matters in AI
Vectorization is the most important performance skill in numerical Python. Machine learning routinely works with arrays of millions to billions of elements. Code that uses Python loops for these computations is too slow for practical use. Vectorized code is also shorter and less error-prone.
Common mistakes
  • Using Python loops over large arrays when a vectorized equivalent exists — always look for a NumPy ufunc or axis operation before writing a loop.
  • Not using keepdims=True when dividing by row or column sums — without keepdims, the sum collapses to a 1D array that broadcasts incorrectly.
  • Accidentally creating very large intermediate arrays via broadcasting — X[:,None,:] - X[None,:,:] creates a (N,N,d) tensor; for N=10,000 and d=100, this is 10^10 floats = 80GB. Be aware of the memory cost before broadcasting over large dimensions.

Matrix operations

Detailed explanation

Matrix operations are the mathematical core of linear algebra and deep learning. The four fundamental operations: addition (A + B, element-wise — requires same shape), scalar multiplication (c * A, multiplies every element by c), matrix multiplication (A @ B, requires A.shape[1] == B.shape[0], outputs shape (A.shape[0], B.shape[1])), and element-wise (Hadamard) multiplication (A * B, same shape — different from matrix multiplication!).

Matrix multiplication computes the dot product of every row of A with every column of B. For A = (m×k) and B = (k×n), the output C = A @ B has shape (m×n). Each element C[i,j] = sum of A[i,:] * B[:,j]. In neural networks, a linear layer computes output = input @ weight.T + bias — a matrix multiplication. Understanding this explains why matrix dimensions must match and how to reshape tensors.

Transpose A.T swaps rows and columns: a (3×4) matrix transposed becomes (4×3). In ML: feature matrices X (n_samples × n_features) are transposed for dot products with weight matrices. Broadcast operations avoid explicit transpose in many cases — NumPy automatically aligns dimensions.

Common linear algebra operations: np.dot(a, b) or a @ b for matrix multiplication, np.linalg.inv(A) for matrix inverse (A must be square and invertible), np.linalg.solve(A, b) to solve Ax=b (more numerically stable than computing A^{-1}b), np.linalg.eig(A) for eigenvalues and eigenvectors, np.linalg.svd(A) for Singular Value Decomposition (used in PCA, dimensionality reduction, and recommendation systems).

In deep learning, almost all computation is matrix multiplication. A forward pass through a neural network is a sequence of: X @ W1 + b1 → activation → @ W2 + b2 → ... The output of each layer is a matrix. Backpropagation computes gradients via matrix multiplications of error signals with weight matrices. GPU hardware (CUDA, tensor cores) is optimized for fast matrix multiplication — this is why GPUs accelerate deep learning.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
A = np.array([[1,2],[3,4],[5,6]])  # (3,2)
B = np.array([[7,8,9],[10,11,12]])  # (2,3)
C = A @ B  # (3,3)
print(C.shape)  # (3, 3)
Linear layer: X = np.random.randn(32, 784)  # batch of 32 images flattened
W = np.random.randn(784, 256); b = np.random.randn(256)
output = X @ W + b  # (32, 256)
Element-wise vs matrix: A * B  # Hadamard product (element-wise, same shape)
A @ B  # matrix multiplication (dot product of rows and columns)
SVD: U, S, Vt = np.linalg.svd(X, full_matrices=False)
X_approx = U[:, :k] @ np.diag(S[:k]) @ Vt[:k, :]  # low-rank approximation
Why this matters in AI
Matrix multiplication is the single most important operation in deep learning. Every forward pass, every gradient computation, every embedding lookup is a matrix operation. Understanding it deeply enables you to reason about computational cost (O(n²k) for (n×k)@(k×n)), memory requirements, and shape errors.
Common mistakes
  • Confusing * (element-wise) with @ (matrix multiplication) — A * B and A @ B are completely different operations. In PyTorch, torch.mul is element-wise; torch.matmul is matrix multiplication.
  • Matrix multiplication order matters — A @ B ≠ B @ A in general. The order encodes the transformation direction.
  • Not checking shape compatibility before multiplication — A (m×k) @ B (k×n) requires the inner dimensions to match: k = k. The output shape is (m, n).

Random numbers

Detailed explanation

Reproducibility in ML requires controlling random number generation. Set seeds at the start of every experiment: np.random.seed(42); random.seed(42); torch.manual_seed(42); torch.cuda.manual_seed_all(42). Without fixed seeds, results vary between runs, making debugging and comparison meaningless. The seed is a number that initializes the random state — the same seed always produces the same sequence of 'random' numbers.

NumPy random: np.random.randn(m, n) samples from a standard normal distribution N(0,1) — used for weight initialization. np.random.rand(m, n) samples from uniform [0,1). np.random.randint(low, high, size) samples random integers. np.random.choice(a, size, replace) samples from a 1D array — used for mini-batch sampling and bootstrap. np.random.permutation(n) returns a random permutation of [0..n-1] — used for dataset shuffling.

The new NumPy Generator API (numpy.random.default_rng) is recommended over the legacy numpy.random functions: rng = np.random.default_rng(seed=42); X = rng.normal(0, 1, (1000, 50)). The Generator API is faster, more statistically sound, and provides thread-safe generation.

Random distributions for ML: normal/Gaussian (weight initialization, noise), uniform (learning rate sampling in random search), Bernoulli (dropout — each neuron is dropped with probability p), categorical (class sampling for imbalanced training), Poisson (count data generation), exponential (time-between-events simulation).

Random splitting: np.random.permutation(n)[:train_size] creates a random train split without replacement. This is more flexible than sklearn's train_test_split when you need custom splits or need to ensure no data leakage (time-based splits). For stratified splits (preserve class balance), use StratifiedKFold or manually stratify by sorting by class and taking every nth sample.

Why this matters in AI
Without seeded random numbers, ML experiments are irreproducible. A model that achieves 87% accuracy cannot be reproduced or improved upon if the results cannot be replicated. Seeds are mandatory for scientific ML practice.
Common mistakes
  • Setting only numpy seed but not torch/random seeds — different libraries have independent random states. Set all of them.
  • Setting seeds inside a loop — seeds set inside a training loop are reset every iteration, making the randomness deterministic within the loop (same shuffling every epoch). Set seeds once at the start.
  • Assuming seeded code is fully reproducible on different hardware/OS — CUDA nondeterminism from parallel operations can cause different results even with seeds. Use torch.use_deterministic_algorithms(True) for full reproducibility (at a performance cost).

Why vectorization matters

Detailed explanation

Vectorization means performing operations on entire arrays rather than individual elements in Python loops. NumPy operations are implemented in compiled C/Fortran code and use SIMD (Single Instruction Multiple Data) CPU instructions to process multiple elements simultaneously. A vectorized operation on 1 million elements takes ~1ms; a Python loop takes ~1 second — 1000× slower. This difference becomes critical in ML where data has millions of rows and thousands of features.

Python loop vs NumPy: to compute the mean of 1 million numbers, a Python loop (for x in data: total += x) takes ~100ms. np.mean(data) takes ~1ms. The reason: each Python operation involves dynamic type checking, memory allocation, and interpreter overhead. NumPy bypasses all of this — the C code operates directly on contiguous memory blocks without per-element overhead.

Broadcasting: NumPy's ability to perform operations on arrays of different shapes without explicit loops. To standardize a (1000, 10) feature matrix: normally requires a loop over columns. With broadcasting: X = (X - X.mean(axis=0)) / X.std(axis=0) — automatically broadcasts the (10,) mean vector across all 1000 rows. Broadcasting rules: dimensions are compared right-to-left; a dimension of 1 can broadcast to any size.

GPU vectorization: PyTorch and TensorFlow tensors run on GPUs, which have thousands of cores (NVIDIA A100: 6912 CUDA cores). GPU operations are even more vectorized than NumPy CPU operations — a matrix multiplication that takes 100ms on CPU takes 1ms on GPU. The key: GPU operations have high latency (data transfer to GPU) but enormous throughput (thousands of cores in parallel). Use GPU for large matrix operations, not small ones.

When NOT to vectorize: avoid premature optimization. For operations that are fundamentally sequential (where each step depends on the previous), vectorization is impossible. For clarity: sometimes a loop is more readable than a vectorized expression. Optimize bottlenecks identified by profiling, not all code. Profile with %timeit or line_profiler to find where time is actually spent.

Worked example

Follow the numeric walkthrough below. It starts with a tiny function or training situation, calculates the important quantity, and then connects that number back to model learning.

import numpy as np
import time
n = 1_000_000
data = np.random.randn(n)
# Python loop
start = time.time()
total = 0
for x in data:
    total += x
print(f'Loop: {(time.time()-start)*1000:.2f}ms, result={total/n:.6f}')
# NumPy vectorized
start = time.time()
result = np.mean(data)
print(f'NumPy: {(time.time()-start)*1000:.2f}ms, result={result:.6f}')
# Broadcasting example
X = np.random.randn(1000, 10)  # 1000 samples, 10 features
means = X.mean(axis=0)          # shape (10,)
stds = X.std(axis=0)             # shape (10,)
X_standardized = (X - means) / stds  # broadcasting: (1000,10) - (10,) = (1000,10)
print(f'Standardized mean: {X_standardized.mean(axis=0).round(5)}')  # ≈ 0
# Vectorize custom functions
from numpy import vectorize
def my_func(x): return x**2 + np.sin(x)  # already vectorized (numpy ops)
# But for functions with conditionals:
def categorize(x): return 'high' if x > 0.5 else 'low'
categorize_vec = np.vectorize(categorize)
result = categorize_vec(np.array([0.1, 0.7, 0.3, 0.9]))
Why this matters in AI
Vectorization is what makes NumPy, pandas, and deep learning frameworks fast enough for practical ML. Without vectorization, training a neural network on 1 million samples would take hours instead of minutes. Understanding vectorization explains why you should use np.dot() instead of a loop, why matrix operations are fast, and why GPU training is so much faster than CPU.
Common mistakes
  • Using Python loops inside NumPy operations — for i in range(len(X)): X[i] = transform(X[i]) defeats the purpose. Use NumPy's ufuncs or vectorize() for element-wise operations.
  • Creating Python lists inside tight loops — list.append() is slow. Use np.zeros() to preallocate and fill with indexing: results = np.zeros(n); results[i] = value.
  • Calling .item() on tensors in training loops — loss.item() extracts a scalar and forces a GPU-CPU synchronization. Only call it for logging, not in every step of a hot loop.
